# 深度写作

本文档是关于全妙深度写作的操作指导。

## **功能概述**

深度写作是一个基于开源的multi-agent框架构建的deep research类型的实例，该功能深度融合客户业务数据，围绕数据分析、工具链建设与输出形态设计，实现研究报告生成的端到端智能化流程，能够自动生成新媒体账号监控日报、行业市场分析报告、专题片脚本等多种内容产品

## **功能入口**

进入[妙笔](https://aimiaobi.console.aliyun.com/?product_code=g_broadscope_media&from=bailian#/home)，功能入口在**内容创作Agent>深度写作**。

进入深度写作页面后，页面中央显示**多智能体协助，产出复杂长文**主题，下方提供任务输入框，可配置**写作信源**、**子Agent**及**方法论**等参数，顶部有**历史记录**和**子Agent资料及管理**入口。

## **功能介绍**

### **配置页**

按照要求进行配置并输入写作主题后，点击即可一键启动写作，下面是配置项介绍：

-   **上传本地素材**
    
    -   上传参考文档，最多10个，支持txt、doc、pdf格式，单文件最大10MB；
        
    -   上传数据表，最多3个，支持csv、xlsx格式，单文件最大10MB；
        
-   **写作信源**
    
    -   默认提供「互联网搜索」，联网搜索时可以添加指定网址；
        
        **说明**
        
        在「互联网搜索」中，您可添加指定网站并开启「限定权威数据源」开关。启用后，深度写作将仅从您信任的站点中检索信息，确保内容素材来源权威、可靠、可控。
        
        在左侧导航栏单击**配置**，选择**写作信源**页签，在**互联网搜索**数据源中打开**指定网址**对话框，输入或批量粘贴目标网址（如 `aliyun.com`、`baidu.com` 等），单击**确认添加**完成配置。
        
    -   添加指定数据源，目前支持两种方式：本地上传文档作为数据源，通过API引入数据源；
        
    -   授权OSS对接数据源。
        
        **说明**
        
        添加指定数据源和授权OSS对接数据源的操作方式可参考《[新建数据源](https://help.aliyun.com/zh/model-studio/ai-miaosou#885a03b827reb)》文档进行配置。
        
-   **子Agent**
    
    -   项目经理：负责规划任务，与用户一起分析任务目标，并将目标拆解成任务分发给其他Agent，并检查每个阶段性成果。
        
    -   数据收集员：负责从互联网搜索，阅读资料，并将有效信息整理成文档，作为后续的分析素材。
        
    -   报告撰写师：负责撰写文档，总结现有材料，形成行文清晰的文档，输出给用户。
        
    -   数据分析员：负责分析数据，包括离线分析数据和在线搜索的数据，并根据分析结果绘制图表。
        
        **说明**
        
        当启用「数据分析员」子Agent 时，系统提供两种图表呈现方式：可选择仅生成图表，或同时输出图表与原始数据表格，满足不同场景下的查看与验证需求。
        
        单击页面右上角的齿轮图标，即可在下拉菜单中选择对应的呈现方式。
        
    
    其中**项目经理**、**数据收集员**和**报告撰写师**为必选子Agent，默认开启且不可关闭；**数据分析员**为可选子Agent，可通过开关自行启用或关闭。
    
-   **方法论**：官方提供案头调研报告、市场分析报告、消费者洞察报告、纪录片脚本的方法论，用户也可以自定义方法论，创作不同类型的报告。
    
-   **目录**：开启目录开关后，生成的报告开头将自动生成目录，点击目录中的章节标题即可快速跳转至对应内容。在输入区域输入创作指令（例如"生成的文章要带有目录"），系统会自动识别意图并在输入框下方显示**目录**开关。开启该开关后，生成的文章将自动包含目录结构。单击右侧发送按钮即可提交任务。
    

### **结果页**

**说明**

深度写作任务需进行深度检索与内容规划，单次生成预计耗时 20 分钟以上。我们已为您开启站内提醒，任务完成后将第一时间通知您，请耐心等待。

任务完成后，页面弹出**深度写作任务已完成**通知，单击**去查看结果**按钮即可查看生成的报告。

1.  写作完成后，以下逐一介绍各模块的功能。
    

-   左侧是项目管理Agent设计的执行计划，各Agent会协同完成任务；写作完成后，结果页左侧展示任务信息：顶部为用户输入的研究需求，中间为**执行计划**模块，列出所有数据收集与分析步骤并以绿色勾选标记完成状态，底部显示本轮研究配置（数据源数量、子Agent数量）以及生成的输出文件（如`final_report.md`），支持下载。
    
-   右侧是结果：
    
    -   报告：展示生成的报告，包含验真视图和发布视图。验真视图允许您在线查验报告的数据与生成依据，完整保留中间文件以支持溯源；而发布视图则是下载后正式报告，删除了中间文件；结果页包含**报告**、**活动**、**文件**、**数据源**四个页签。其中**报告**页签下设有**验真视图**和**发布视图**两个子视图，用于查看生成报告的摘要内容，包括分析目的与范围、预测逻辑等章节及其引用的数据源标签。
        
    -   活动：任务的思考和执行过程；**活动**：展示任务执行过程中各角色Agent（如项目经理、数据收集员）的协作活动流程，包括任务分配、思考分析、联网搜索及中间文件写入等操作步骤，便于追溯报告生成的完整工作过程。
        
    -   文件：任务执行过程中生成的中间文件，基于检索到的数据源进行整理与摘要后形成，支持在线预览与下载，并可追溯原始搜索数据源。**文件**：展示任务过程中生成和使用的文件列表，您可以对单个文件进行预览和下载操作。在验真视图中，单击中间文件（如 `market_forecast_industry_growth_saas.md`）可打开弹窗查看文件内容。弹窗下方的**原始搜索结果**区域展示数据来源信息，包括 URL、发布时间、站点名称和标题，便于数据溯源。
        
    -   数据源：生成当前报告所检索的数据源。**数据源**：展示任务搜索并阅读的所有网页来源列表，包括来源类型标识和对应的 URL 链接，便于追溯报告数据的原始出处。
        

2.  **溯源验真**：在验真视图下，所有生成内容均关联可追溯的中间文件。您可通过中间文件查看其原始来源，实现完整溯源与真实性验证。
    
    报告正文中以内联方式引用中间文件名（如 `market_scope.md`），单击可查看对应内容；右侧边栏展示编号的原始数据来源列表，支持逐条追溯。
    
3.  **AI导读**：系统为每篇报告自动生成内容概述、关键要点、关键词、速读摘要和思维导图，并支持基于报告内容的问答功能，让您读得更快、理解得更深、问得更准。
    
    深度写作结果页分为左右两栏：左侧为任务执行面板，顶部显示用户输入的 prompt，下方**执行计划**列出各步骤及其完成状态；底部显示生成的文件（如 `final_report.md`）及**AI导读**入口按钮。右侧为报告内容预览区。AI导读界面采用左右分栏布局，左侧展示报告正文内容，右侧面板以结构化方式呈现系统生成的摘要信息及交互入口。
    
4.  **报告下载**：支持导出为 Markdown、PDF 或 ZIP 压缩包（含完整资源）三种格式。
