# 通用支持与应用支持对比

本页对比阿里云百炼平台的两类支持文档：**通用支持（[support](../guides/support.md)）** 与 **应用支持（application [support](../guides/support.md)）**。前者面向平台整体使用，覆盖计费、账号、模型训练、幻觉优化等通用问题；后者聚焦应用构建、插件、RAG、数据管理与合规备案等应用侧能力。开发者在技术选型与排障时，可根据问题归属选择对应支持入口，避免跨域查找。

## 关键维度对比

| 维度 | 通用支持 ([support](../guides/support.md)) | 应用支持 ([application support](../guides/application-support.md)) |
|------|--------------------|-------------------------------|
| 关注层级 | 平台级（账号、计费、模型训练、API/SDK 基础调用） | 应用级（插件、Agent/Assistant API、RAG、数据管理） |
| 计费视角 | 模型调用单价、模型部署与训练计费、预付费/后付费、节省计划与资源包、账户余额与开票 | 自定义插件服务暂不收费；智能体 API 的 [prompt](../guides/prompt.md) 优化、应用调用与测试产生费用 |
| API/SDK 能力 | Completion API 与 Assistant API 两种调用方式；Java/Python SDK；错误码（如 `100004` 参数缺失） | Agent 与 Assistant API 的区别（Agent 侧重插件调整与上下文理解，Assistant API 提供封装类便于调优）；自定义插件不支持自定义 header，仅支持 `authorization` |
| 模型与训练 | 文字与图片训练；`qwen-vl-plus` 支持图片微调；`qwen-turbo` 重速度低费用、`qwen-max` 重精度；千问支持 14 种语言；本地/开源模型不支持上传与导出 | 偏重模型在应用中的调用与表现优化，未覆盖训练细节 |
| 幻觉优化 | 明确降幻觉策略：选更强模型（Max > Plus > Turbo）、提示词工程、RAG、插件/MCP、降低 `temperature`/`top_k`/`top_p`、后处理验证 | RAG 回复准确性优化：问题反馈按钮、RequestId 工单反馈；流式与增量输出 (`stream=True` / `incremental_output=True`)；Markdown 渲染 `**xxx**` |
| 插件能力 | 未展开插件清单 | 六款官方插件：Python 代码解释器、计算器、图片生成、夸克搜索、生成二维码、GitHub 搜索；部分需申请；自定义插件遵循协议传给大模型理解 |
| 数据管理 | 强调数据安全与合规：AES-256 加密、不用于训练、调用数据存储条款 | 具体限制：文件格式 PDF/DOC/DOCX（PDF 后缀必须小写）、[业务空间](../concepts/workspace.md)最多 10 万文档、MD5 校验必填、结构化数据表格不能有空行 |
| 账号与业务隔离 | 按地域开通；实名认证；主账号给子账号授予不同[业务空间](../concepts/workspace.md)权限实现隔离；模型体验中心最多 100 条历史记录；无独立手机应用 | 以[业务空间](../concepts/workspace.md)为数据管理单位，依赖通用支持中的业务空间权限体系 |
| 合规备案 | 数据安全与合规条款、应用合规备案流程入口、千问合作协议需提工单申请 | 应用合规备案落地步骤：先按指南备案，再通过工单申请通义千问系列模型合作协议 |
| 服务协议 | 阿里云百炼服务协议等条款清单（协议链接被截断，需查原始协议页） | 明确列出三项：阿里云百炼服务协议、服务特别说明、开源模型协议条款说明 |
| 典型入口 | 控制台模型市场、API-Key 管理、费用与成本、发票管理、模型体验中心 | 应用构建页、插件管理、RAG 知识库、应用合规备案指南 |

## 适用场景建议

- **选通用支持 (support)**：当你需要处理账号开通、地域切换、API-Key 创建与删除、模型调用计费查询、模型训练与微调选型、降低模型幻觉、查看服务协议与数据安全条款等平台级问题时。这类问题与具体应用无关，属于平台底座。
- **选应用支持 ([application support](../guides/application-support.md))**：当你在构建具体应用时遇到插件配置、Agent 与 Assistant API 选型、RAG 检索模式与回复优化、[流式输出](../concepts/streaming-output.md)、知识库文档上传与结构化数据限制、应用合规备案落地等应用侧问题时。
- **两者协同**：应用侧的 RAG 准确性、插件调用、数据管理依赖平台侧的业务空间权限、API-Key 与计费体系；应用合规备案的协议基础则来自通用支持中的服务协议清单。建议先在通用支持确认账号与协议基础，再在应用支持落地应用能力。

## 技术选型参考

1. **调模型还是调应用**：仅做模型推理补全用 Completion API（通用支持范畴）；需要在应用内编排插件、上下文与 RAG，走 Agent 或 Assistant API（应用支持范畴）。
2. **是否自训模型**：需要领域微调选 `qwen-vl-plus`（图片）或千问文本模型，并参考通用支持的训练数据与超参实践；纯应用层调用则无需训练，直接走应用支持的 RAG 与插件能力。
3. **成本控制**：通用支持提供节省计划、资源包与后付费出账节奏；应用支持明确插件服务暂不收费，但智能体 API 调用会产生 [prompt](../guides/prompt.md) 优化与测试费用，两者结合做整体预算。
4. **合规优先级**：若产品需上架小程序或应用市场，先在通用支持确认数据安全与协议条款，再按应用支持的备案步骤落地。
