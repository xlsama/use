# plug in

插件是阿里云百炼平台中用于扩展大模型能力的核心机制。通过将插件集成到应用中，可以弥补大模型在实时信息获取、精确计算、代码执行等方面的不足。百炼支持官方插件、第三方插件和自定义插件三种类型，开发者可根据业务需求灵活选择或自行开发。

## 插件类型

### 官方插件

百炼组件广场预置了多个官方插件，无需配置即可直接调用。主要包括：

| 插件名称 | 工具 ID | 功能说明 | 计费 |
|---------|---------|---------|------|
| Python 代码解释器 | `code_interpreter` | 执行 Python 代码，支持数学计算、数据分析与可视化 | 免费 |
| 计算器 | `calculator` | 复杂数学计算 | 免费 |
| 图片生成 | `text_to_image` | 基于文本生成图片 | 限时免费，需申请开通 |
| 夸克搜索 | `quark_search` | 搜索实时信息（返回网页标题、关键词和摘要，不支持访问网页详情） | 限时免费，需申请开通 |
| 生成二维码 | `generate_qrcode` | 根据 URL 生成二维码 | 免费 |
| GitHub 搜索 | `github_search` | 搜索 GitHub 项目列表 | 免费 |

详细的官方插件使用说明和效果示例，请参见 [官方和第三方插件](../../raw/application-user-guide/plug-in/plugins.md)。

### 第三方插件

第三方插件涵盖商业服务、图像视频、学习教育等领域，经过效果测试，开通后可直接调用，无需额外配置。开通流程：在插件页面找到目标插件 → 选择套餐 → 免费试用或购买。

### 自定义插件

当官方和第三方插件无法满足需求时，可通过创建自定义插件来扩展大模型能力。支持两种创建方式：

- **个性化开发**：手动创建插件并配置 API 路径、请求参数、鉴权信息等。完整流程为：创建插件 → 添加工具 → 调试与发布 → 在应用中使用。
- **从云市场导入**：直接导入已在阿里云云市场开通的 API 作为插件，系统自动填充参数信息。

自定义插件的完整开发流程请参见 [自定义插件](../../raw/application-user-guide/plug-in/custom-plug-ins.md)。

## 支持的模型

| 模型 | 模型标识符 |
|------|-----------|
| 通义千问-Turbo | `qwen-turbo` |
| 通义千问-Plus | `qwen-plus` |
| 通义千问-Max | `qwen-max` |
| 通义千问VL-Max | `qwen-vl-max` |
| 通义千问VL-Plus | `qwen-vl-plus` |

> **注意**：各模型对插件的兼容性可能存在差异，最新兼容性状态以控制台实际执行结果为准。

## 插件调用机制

调用插件的本质是调用插件下的工具。百炼支持通过以下三种方式调用插件：

1. **智能体应用**：在应用编排页面添加插件工具，大模型根据用户输入、工具名称和描述自动判断是否调用。
2. **工作流应用**：将插件作为工作流节点，按编排方式执行特定任务（非大模型主动规划）。
3. **Assistant API**：通过 `tools` 参数在 API 请求中指定工具。

调用流程：大模型判断是否需要工具 → 选择合适的工具 → 应用内部完成调用 → 将工具返回结果与用户内容合并 → 大模型生成最终结果。如无需调用工具，大模型直接生成结果。

更多关于插件调用机制的说明，请参见 [插件概述](../../raw/application-user-guide/plug-in/plug-in-overview.md)。

## 自定义插件关键配置

### 鉴权配置

自定义插件支持服务级鉴权和用户级鉴权，鉴权信息可放在 Header 或 Query 中：

- **Header**：鉴权信息放在 HTTP 请求头的 Authorization 字段中
- **Query**：鉴权信息附加在 URL 参数中

Token 类型支持 `basic`（不加前缀）、`bearer`（自动添加 "Bearer" 前缀）、`appcode`（自动添加 "APPCODE" 前缀）。

### 工具参数配置

输入参数的传参方式分为两种：

- **大模型识别**：参数值由大模型从用户输入中自动提取
- **业务透传**：参数值从外部主动传入，不经过大模型处理，通过 `biz_params` 和 `user_defined_params` 传递

### 发布为 MCP 服务

自定义插件支持发布为 MCP 服务，然后在智能体应用的 MCP 区块中添加使用，提供更灵活的集成方式。

## 使用限制与注意事项

- 每个智能体应用最多支持添加 10 个工具
- Python 代码解释器不支持对外访问网络及上传本地文件
- 夸克搜索仅支持检索网页标题、关键词和摘要，不支持访问网页详情
- 官方插件只能与位于相同[业务空间](../concepts/workspace.md)的智能体应用关联
- 从云市场导入的插件为草稿状态，需测试并发布后才能使用
- 删除插件或工具会导致关联应用失效，操作不可撤回

## 常见问题

**夸克搜索和联网搜索（enable_search）有什么区别？**

- **夸克搜索插件**：模型直接调用插件执行搜索，将搜索结果以文本形式返回，支持检索网页标题、关键词和摘要，不支持访问网页详情。
- **联网搜索（enable_search）**：同样基于夸克搜索，但模型会尝试利用互联网信息来丰富生成内容，不会完全依赖或返回搜索结果。
