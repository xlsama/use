# llm application

阿里云百炼平台提供三种核心应用构建模式来突破大模型的原生局限：**智能体（Agent）应用**由提示词驱动自主决策调用工具，适合开放式对话；**工作流（Workflow）应用**通过可视化节点编排固定流程，适合稳定可复现的多步骤任务；**高代码应用**面向专业开发者基于 Python 代码部署 AI 后端服务。三类应用都能集成知识库、MCP、记忆等核心能力。

## 应用类型对比

百炼平台的三种应用在开发方式、特点和适用人群上有明显差异，开发者可参考[应用类型介绍](../../raw/application-user-guide/llm-application/application-introduction.md)进行选型：

| 对比维度 | 智能体（Agent） | 工作流（Workflow） | 高代码应用 |
| --- | --- | --- | --- |
| 开发方式 | 自然语言配置（零代码） | 可视化节点编排（低代码） | Python 编码（专业代码） |
| 核心特点 | AI 自主决策、动态规划 | 预定义流程精确控制 | 完全由代码控制逻辑 |
| 适合人群 | 业务人员、产品经理、运营 | IT 运维、业务分析师 | AI 工程师、开发者 |
| 开发门槛 | 低 | 中 | 高 |

### 快速选型

- **智能体（Agent）应用**：由提示词驱动，自主理解用户意图并调用知识库、MCP 等外部工具完成任务。适合智能客服、知识问答、任务助理、旅行规划等开放式对话场景。
- **工作流（Workflow）应用**：通过可视化节点编排将复杂任务串联成稳定可控的执行链路。适合自动化报告生成、订单处理、多步骤审批流、数据标注等固定流程自动化场景。
- **高代码应用**：基于完整 Python 项目结构部署 AI 后端服务，支持 Serverless Function 和 K8s 两种部署方式。适合部署私有算法、集成复杂系统、构建专业级 AI 后端服务等需要深度定制的场景。

## 智能体应用

智能体应用是百炼的核心应用类型，分为**新版智能体（Agent 2.0）**和**旧版智能体（Agent 1.0）**两个版本。

### 新版与旧版差异

新版智能体（Agent 2.0）将知识库、MCP 等多种能力统一为工具，由智能体自主规划在何时、以何种顺序调用，并能完整展示每一轮"规划-执行-反思"全过程。旧版智能体仅展示最终结果，且智能体在检索知识库后再决策是否调用其他工具，适用于意图单一、流程固定的简单任务。在没有旧版本依赖时，推荐使用新版，详情参见[新版智能体应用](../../raw/application-user-guide/llm-application/new-single-agent-application.md)。

> **注意**：旧版智能体和新版智能体基于不同的技术架构，彼此不兼容，无法进行直接的版本切换、升级或降级。如需体验新版功能，需重新创建一个新版智能体应用。

### 核心能力

新版智能体支持以下能力配置：

- **模型选择**：推荐选用具备强工具调用能力的模型（如`千问-Max`系列）以确保多步规划效果。可配置最长回复长度、temperature、enable_thinking 等参数。
- **系统提示词**：定义智能体角色、行为指令与能力边界，支持嵌入自定义变量。
- **预解析文件**：控制上传文件的处理方式。关闭预解析时文件 URL 作为上下文信息传递给智能体；开启预解析时由预置解析器处理文件返回文本内容。
- **知识库**：作为工具由智能体自主规划调用，支持通过标签限定查询范围。
- **MCP**：外部工具以 MCP 协议接入，包括官方 MCP 和自定义 MCP 服务。智能体可在多步推理中动态、非固定顺序调用。
- **应用组件**：将已发布的智能体或工作流作为工具接入。
- **记忆**：短期记忆支持 0 到 30 轮上下文；长期记忆计划在后续迭代支持。
- **回复**：支持展示回答来源，以角标形式展示知识来源和源文件地址。

### 关键参数

- **ReAct 最大轮次**：取值范围 1-50，限制智能体单次会话中调用工具的最大次数。超出限制后自动退出工具调用链路并生成最终回复。

### 运行与结果分析

新版智能体以卡片流形式展示决策过程，包含两种步骤：

1. **思考（Thinking）**：展示模型推理逻辑，便于分析决策路径（仅当选用支持思考模式的模型时出现）。
2. **工具调用**：记录模型执行的工具调用入参及返回结果。

## 工作流应用

工作流应用通过节点编排将复杂任务拆分成有序步骤，详细用法参见[工作流应用](../../raw/application-user-guide/llm-application/workflow-application.md)。

### 节点类型

节点是工作流的核心功能单元，主要节点包括：

- **开始/结束节点**：定义输入/输出参数。开始节点预置 `query`（用户输入）、`historyList`（对话历史）、`imageList`（图片列表）等变量。
- **大模型节点**：配置模型、提示词、用户提示词和记忆。记忆支持"本节点缓存"（仅当前节点聊天内容）和"自定义缓存"（全局聊天内容）。
- **意图分类节点**：根据用户输入分发到不同分支，常用于多意图场景路由。
- **变量处理节点**：执行文本输出等数据处理操作。
- **智能体群组节点**：将任务分解为多个子智能体协作完成，每个子智能体需添加描述信息帮助决策模型判断调用时机。

### 会话变量

会话变量作为全局变量，能够在当前工作流的全生命周期内记录参数信息，并可在各个节点中引用，可在画布配置页面右上角进行配置。

### 典型案例

- **识别诈骗信息**：开始 → 大模型 → 结束，提示词中明确审查关键词汇和典型诈骗模式。
- **智能导购**：开始 → 意图分类（电视/冰箱/手机）→ 多个大模型节点 → 结束，每个大模型节点按参数列表逐步询问。
- **日程管理助手**：使用智能体群组节点，将任务分解为信息收集和数据整理两个子智能体协作完成。

## 高代码应用

高代码应用允许开发者基于完整的 Python 项目结构部署 AI 后端服务，详细介绍参见[高代码应用](../../raw/application-user-guide/llm-application/rich-code-application.md)。

### 部署方式

- **Serverless Function**（默认）：适用于低负载、无状态、快速拉起的场景，部署花销低。
- **K8s**：适用于高性能、有状态、执行长程任务的 Agent，使用前需开通 ACK 容器服务并完成授权。

> 时延敏感业务建议最小实例数 ≥ 1，可实现毫秒级热启动、保障服务不中断。

### 创建方式

- **控制台创建**：在控制台直接选择模板、配置资源并一键部署，推荐初次使用。
- **命令行创建**：在控制台创建空白应用后，通过 AgentScope-AI 命令行工具上传本地代码包部署，适合已有项目代码的开发者。

### 应用模板

| 模板 | 说明 |
| --- | --- |
| 基础对话 Agent | 以 LLM 对话为主的基础示例，适合快速体验和二次开发 |
| 工具调用 Agent | 内置多种 MCP 工具（联网搜索、计算器等），展示工具调用能力 |
| 深度研究 Agent | 基于 AgentScope 组件搭建，可进行互联网调研并生成研究报告 |

### 管理功能

应用详情页提供四个功能 Tab：

- **部署**：管理部署状态、资源配置、运行日志，支持重新部署和停止服务。
- **工具**：一站式 MCP 工具接入，可关联知识库、工作流、插件等 MCP 服务。
- **网关**：通过自定义域名和网关路由在生产环境访问云上服务。应用部署地域应与网关所在地域相同。
- **前端**：提供直接体验、自定义交互卡片和自定义前端 WebUI（基于 [Spark Design](https://github.com/agentscope-ai/agentscope-spark-design) 框架）三种方式。

> 生产环境推荐打开测试域名的"禁止公网访问"开关，测试用触发器将不再能够通过公网访问。

## 文件问答

文件问答是智能体应用中的文件处理能力，支持对文档、图片、音视频等多种文件进行内容理解和智能问答，详细介绍参见[文件问答](../../raw/application-user-guide/llm-application/file-q-a.md)。

### 处理模式

| 模式 | 适合场景 | 特点 |
| --- | --- | --- |
| **全文引用** | 文档总结、全文翻译、风格润色 | 简单直接，受限于模型上下文长度 |
| **切片检索**（RAG） | 长文档问答、知识库检索、精确信息定位 | 能处理超长文件，效果依赖切片和检索策略 |
| **自定义处理** | 图片风格转换、视频内容分析后生成报告 | 功能强大灵活，依赖所配置的工具 |

### 文件格式与限制

- 单个会话上传文件上限 **10 个**，单文件不超过 **10 MB**。
- 支持文档（`.doc`/`.docx`/`.pdf` 等）、图片（`.png`/`.jpg` 等）、视频（`.mp4`/`.mov` 等）、音频（`.mp3`/`.wav` 等）。
- 超过 10 MB 的文件推荐使用文件上传 API（`session_file_id`），生产环境尤其推荐此方式。

> **注意**：通过聊天窗口上传的文件仅在当前会话中有效，刷新或关闭页面将导致文件丢失。通过文件上传 API 上传的文件有效期通常为 24 小时。

### 全文引用参数

- **单文件最大解析长度（token）**：限制单个文件提取的 token 数量，超出部分从文件末尾截断。
- **最大拼装长度（token）**：限制所有文件内容拼接后的总 token 数量，超出部分从最后拼接文件的末尾开始截断。

### 切片检索参数

- **召回片段数**：模型回答时引用的相关文本片段的最大数量。
- **最大拼装长度**：限制所有被召回片段拼接后的总 token 数量。若超出，系统将根据相关性得分从低到高依次丢弃召回片段。

### 自定义处理的图片处理

当选择千问 VL 系列模型时，图片文件支持两种处理方式：

- **模型处理**：仅使用视觉能力直接分析图片，不调用外部工具，适用于看图问答。
- **模型处理 + 规划**：模型看懂图片后判断是否调用外部工具，适用于需要编辑、转换或借助工具分析图片的场景。

> 千问-VL 系列模型由于具备多模态能力，即使关闭预解析文件，也能够直接解析图片和视频文件。

### API 调用限制

- 文件按智能体应用中保存的配置进行处理，API 调用时无法动态切换处理模式。
- 默认调用频率上限 **100 次/分钟**，此限制共享该应用的所有 API 请求配额。

## 应用发布与集成

### 发布前提

应用发布是后续所有调用和集成的前提条件。在应用配置页面右上角单击**发布**，弹窗会展示自上次发布以来的配置变更差异，确认后完成发布。

> 若应用由 RAM 账号创建，发布前请确认已拥有服务关联角色权限 `ram:CreateServiceLinkedRole`。

### API 调用

在应用**发布渠道**页签单击**API调用**右侧的**查看API**查看调用方法。需替换为实际的百炼 API Key 后方可发起调用。

### 版本管理

支持编辑历史版本描述信息或回滚发布过的历史版本。选中目标历史版本后，可单击**覆盖当前草稿**回滚至该版本（该版本内容将覆盖当前草稿）。

## 计费说明

应用计费主要涉及以下几个方面：

1. **模型调用**：根据模型类型和 Token 用量计费。不同文件处理模式下输入 Token 消耗不同：全文引用消耗最大，切片检索通常远小于全文引用，自定义处理取决于工具交互复杂性。
2. **知识库**：按量付费，召回的文本切片会增加模型输入 Token 数量。
3. **MCP**：部分官方 MCP（如文生图、文生视频、语音合成）按模型调用计费；部分 MCP 涉及第三方 API 调用，费用由第三方收取，百炼不另行收费。
4. **长期记忆**：数据存储不收费；记忆体内容占用 Token 暂不计费，但会合并到 Prompt 中增加 Token 消耗。
5. **文件上传**：上传操作本身不收费。

> 仅创建应用不收费。调用应用进行问答时，根据所用模型类型收取模型调用费用。

## 限制与注意事项

- 旧版与新版智能体不兼容，无法直接升级或降级，需重新创建。
- 自定义插件超时限制为 **5 秒**。
- 文件问答功能仅适用于中国大陆版（北京地域）。
- 模型支持情况以智能体应用内显示为准，数据更新可能存在延迟。
- 知识库检索的文本会占用模型上下文窗口，需根据实际情况调整检索策略和文本长度以避免超限。
- 高代码应用部署后将开始计费，函数、网关、存储、模型调用均会产生费用。
- 若应用对外提供服务，必须完成应用合规备案。

## 常见问题

**为什么智能体未按预期调用已配置的工具？** 可从四个层面排查：技能是否已正确挂载、系统提示词是否清晰描述技能功能与适用场景、用户意图是否清晰指向特定技能、是否已达到 ReAct 轮次上限。

**配置了知识库但回答与知识库无关如何解决？** 先进行知识库命中测试查看相似度得分；在提示词中明确要求模型仅基于知识库回答；必要时更换模型。

**是否支持通过 API 创建智能体应用？** 支持使用 Assistant API 创建功能类似智能体的大模型应用，但 Assistant API 创建的应用不支持控制台管理。
