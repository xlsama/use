先使用 `EnterPlanMode` 进入计划模式，确保修改前充分理解问题。

## 问题分类

| 类型     | 特征                       | 处理方式           |
| -------- | -------------------------- | ------------------ |
| Bug 修复 | 运行时错误、逻辑缺陷       | 定位根因，修复代码 |
| 需求调整 | 功能与期望不符、需新增功能 | 确认变更，评估影响 |

类型不明确时，用 `AskUserQuestion` 询问用户确认。

## 问题分析

对每个问题输出：

- **类型**：Bug / 需求调整
- **现象**：用户观察到的异常行为
- **根因**：代码层面的问题原因
- **涉及文件**：`path/to/file.ts:行号`
- **解决方案**：具体修复步骤

## 逐个修复

按优先级逐项修复并验证；多问题时用 `TaskCreate` 跟踪进度。
