# 千问大模型应用上架及合规备案

各位开发者开发的应用/小程序中通过阿里云百炼大模型服务平台接入了千问、万相等系列大模型，想要将接入大模型的应用/小程序上架到应用市场/小程序平台中，上架平台需要开发者提供各种资料，本篇将引导各位开发者如何获取对应资料。

## 背景及法规依据

根据2023年8月15日生效的《[生成式人工智能服务管理暂行办法](https://www.moj.gov.cn/pub/sfbgw/flfggz/flfggzbmgz/202401/t20240109_493171.html)》要求，监管部门对AIGC相关的APP和小程序进行盘点整改，未完成合规手续的应用将被下架，未上架的应用必须完成合规动作后方可上线。

阿里云百炼大模型服务平台面向云上企业提供相关大模型技术支持，涉及不同应用方的使用场景差异，可能涉及合规要求的算法备案，备案中涉及算法相关研发信息及内容，因此我们汇总整理了支持云上企业查询大模型算法备案的指南。

**重要**

-   上架及合规备案前，请开发者仔细阅读[《生成式人工智能服务管理暂行办法》](https://www.moj.gov.cn/pub/sfbgw/flfggz/flfggzbmgz/202401/t20240109_493171.html)原文或咨询专业人士。相关法律法规、监管政策及平台要求可能发生变化，建议开发者定期关注最新规定，根据自身具体情况咨询专业法律顾问，独立判断并承担相应的法律责任。
    
-   本文仅提供备案参考及所需材料获取方法，不构成法律意见，即使使用了阿里云提供的模型及备案信息，应用/小程序开发者仍是《生成式人工智能服务管理暂行办法》等法规定义的“服务提供者”，需履行内容审核、用户保护、数据安全、标识规范等全部法定义务，开发者主体需独立承担其应用/小程序的全部法律责任。
    

## **需要备案的典型场景示例**

以下场景仅为示例，实际大模型应用上架中需要备案的场景包含但不限于以下几类：

-   **场景1：**面向C端用户且**不具有**舆论属性或社会动员能力的场景
    
    **场景示例**：个人助手类工具（如智能客服、学习辅导APP）、娱乐类应用（如AI生成表情包、小说续写工具）。
    
    **需提供材料**：
    
    -   大模型算法[备案信息](#6105669eaa7jm)。
        
    -   应用主体与阿里云的[合作协议](#d48d90a6008rg)。
        
-   **场景2：**面向C端用户且**具有**舆论属性或社会动员能力的场景
    
    **重要**
    
    “具有舆论属性或社会动员能力”的界定可能较为复杂，建议开发者结合自身应用/小程序的具体功能，仔细评估，必要时请咨询专业法律顾问或向相关监管部门。
    
    **场景示例**：社交媒体内容生成工具（如自动生成新闻摘要、评论区AI回复功能）、舆情分析类应用（如热点事件追踪工具）。
    
    **需提供材料**：
    
    -   场景1所需的全部材料。
        
    -   企业自主完成安全评估报告（具体要求参见法规[《具有舆论属性或社会动员能力的互联网信息服务安全评估规定》](https://www.gov.cn/zhengce/zhengceku/2018-11/30/content_5457763.htm)）。
        
    -   企业自主完成算法备案（具体要求参见法规[《互联网信息服务算法推荐管理规定》](https://www.gov.cn/zhengce/zhengceku/2022-01/04/content_5666429.htm)）。
        
-   **场景3：**企业内部使用场景
    
    **场景示例**：企业内部的智能文档处理工具、员工培训问答系统（无需对外提供服务）。
    
    **说明**
    
    对于此场景，虽然可能不直接适用《生成式人工智能服务管理暂行办法》关于向公众提供服务的要求，但开发者仍需关注数据安全、保密等内部合规要求。阿里云原则上不为此类内部应用场景提供专门的合规咨询支持，但开发者如因内部管理或未来业务扩展需要，仍可按本文查询并记录所使用模型的备案信息。
