# 视频生成与编辑

选择适合文生视频、图生视频、视频编辑等场景的模型。

## 文生视频

通过文本提示词生成有声视频，推荐使用`happyhorse-1.0-t2v`。支持1080P分辨率，单片段最长15秒。

### 需要传入自定义音频文件

如果需要传入自定义音频文件（如自定义旁白、配乐），推荐使用`wan2.7-t2v-2026-04-25`。

## 图生视频

将静态图片转化为动态视频。首帧生视频推荐使用`happyhorse-1.0-i2v`，首尾帧生视频推荐使用`wan2.7-i2v-2026-04-25`。

### 首帧生视频

基于单张图片生成视频，推荐使用`happyhorse-1.0-i2v`（支持音频、1080P、3至15秒）。如果需要传入自定义音频文件，推荐使用`wan2.7-i2v-2026-04-25`。

### 首尾帧生视频，构建长视频

使用首尾帧模型（推荐`wan2.7-i2v-2026-04-25`）可以串联多个片段：将上一片段的末帧作为下一片段的首帧，实现无缝过渡，适用于叙事、产品演示或教程等场景。

## 参考生视频

基于参考图片在不同场景中保持角色一致性，推荐使用`happyhorse-1.0-r2v`。如果需要传入自定义音频文件定义音色，或传入视频参考主体，推荐使用`wan2.7-r2v`。

## 视频编辑

通过文本指令对已有视频进行风格转换、元素替换等编辑操作，推荐使用`happyhorse-1.0-video-edit`。如果需要进行特效复刻、运镜复刻，推荐使用`wan2.7-videoedit`。

## 角色动画

### 基于动作参考驱动人物动画

将参考视频中的动作迁移到静态图片中的人物上，推荐使用`wan2.2-animate-move`。背景保持不变。专业模式（wan-pro）更接近真实拍摄效果，标准模式（wan-std）速度更快、成本更低。

### 视频中替换人物

将视频中的人物替换为图片中的人物，推荐使用`wan2.2-animate-mix`。同样支持pro和std两种模式。
