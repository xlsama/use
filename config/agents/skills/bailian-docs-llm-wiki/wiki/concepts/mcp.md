# 模型上下文协议

模型上下文协议（Model Context Protocol, MCP）是一种开源标准协议，用于在大模型与外部工具之间建立统一的信息传递通道。在阿里云百炼平台中，MCP 是扩展智能体和工作流应用工具调用能力的核心机制，开发者无需为每个外部工具编写专用接口即可接入海量第三方能力。

## 在百炼平台的使用场景

MCP 在百炼中贯穿三类应用场景：应用内工具调用、自定义服务部署、以及通过外部调用集成到第三方应用。

### 应用内工具调用

- **智能体应用**：大模型根据用户输入自动判断是否调用 MCP 服务，一个智能体最多可同时添加 5 个 MCP 服务，支持单 MCP 和多 MCP 协同调用。新版智能体（Agent 2.0）将 MCP 与知识库等能力统一为工具，由智能体自主规划调用时机和顺序；旧版智能体则先检索知识库再决策是否调用其他工具。
- **工作流应用**：每个 MCP 节点只能使用一个工具，需手动指定输入参数并将输出传递到下一个节点。典型链路为大模型节点（提取参数）→ MCP 节点（调用工具）→ 大模型节点（整理结果）→ 输出节点。
- **高代码应用**：基于 Python 代码部署的 AI 后端服务，也可集成 MCP 等核心能力。

### MCP 服务类型

- **官方 MCP 服务**：由百炼官方部署在云端，开通即用，包括 Amap Maps、Sequential Thinking、QuickChart、Firecrawl、联网搜索等。
- **自定义 MCP 服务**：支持开发者部署自有或开源社区的 MCP 服务，托管在函数计算 FC 上。

### 自定义部署方式

| 部署方式 | 适用场景 | 安装方式 |
|---------|---------|---------|
| 使用脚本部署 | 拥有 MCP 协议代码包（自研或开源） | npx（Node.js）/ uvx（Python）/ http（远程） |
| 从 AI 网关导入 | 已有非 MCP 规范的 RESTful API | 通过 AI 网关将 API 升级为 MCP 服务 |
| 从阿里云 OpenAPI 导入 | 操作阿里云产品（OSS、ECS 等） | 通过 OpenAPI 开发者门户发布 |

脚本部署的配置代码示例：

```json
{
  "mcpServers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

### 外部调用集成

百炼 MCP 服务支持通过外部调用集成至第三方应用或个人项目：

- **一键配置到第三方应用**：支持 Cherry Studio 和 Cursor，自动或手动导入配置。
- **通过 SDK 开发集成**：使用 MCP SDK（如 Qwen Agent 框架）编码调用，通过 Streamable HTTP 协议连接。

## 关键参数和配置

### 外部调用端点

调用 URL 格式为 `https://dashscope.aliyuncs.com/api/v1/mcps/<service-name>/mcp`，需在 Headers 中携带百炼 API Key 进行鉴权。百炼 MCP 服务已从旧版 SSE 协议升级为 Streamable HTTP 协议，已开通用户需取消开通后重新开通以完成升级。

### 协议类型与端点匹配

配置中 `type` 字段必须与端点路径匹配：`"sse"` 对应 `/sse` 路径，`"streamableHttp"` 对应 `/mcp` 路径。配置错误会触发 `MCP_PROTOCOL_ERROR`。

### 智能体调用参数

- **ReAct 最大轮次**：取值范围 1-50，限制智能体单次会话中调用工具的最大次数。
- 推荐选用具备强工具调用能力的模型（如 `qwen-max` 系列）以保障多步规划效果。

## 与插件的关系

MCP 与插件是百炼扩展大模型能力的两种互补机制。插件偏向于通过 API 定义扩展单点能力，支持通过 `tools` 参数在 Assistant API 中调用；而 MCP 提供标准化的工具协议通道。两者可以互通：自定义插件支持发布为 MCP 服务，然后在智能体应用的 MCP 区块中添加使用，从而获得更灵活的集成方式。

## 使用限制

- 智能体最多同时添加 5 个 MCP 服务。
- 工作流中每个 MCP 节点只能使用一个工具。
- MCP 服务不能在直接调用千问 API 时接入，必须通过智能体或工作流应用使用。
- 自定义 MCP 服务托管在函数计算 FC，无固定出口公网 IP，访问远程资源需配置 IP 白名单或 VPC 打通。
- 自定义服务无法访问用户本地资源（文件、硬件等）。
- 通过 npx/uvx 部署的服务，上游版本更新后需手动重新部署。
- 调用 MCP 会增加模型的输入和输出 Token 消耗。
- 私有 npm 仓库中的 MCP Server 暂不支持部署，需发布到公共仓库或改用 http 连接。

## 排查要点

- **无法连接**：确认已开通/升级服务、API Key 有效、额度未用尽。
- **模型不调用 MCP**：在提示词中明确指定工具名称和能力描述。
- **部署失败**：确认本地可运行、服务可云端托管、配置代码正确、函数计算权限已开通。
- **协议错误**：确认 `type` 与端点路径匹配（`sse`↔`/sse`，`streamableHttp`↔`/mcp`）。
