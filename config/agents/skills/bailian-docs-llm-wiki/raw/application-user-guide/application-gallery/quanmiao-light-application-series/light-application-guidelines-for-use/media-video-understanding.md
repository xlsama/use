## 功能概述

影视传媒视频理解轻应用通过整合视频处理、自动语音识别（ASR）、视觉语言模型（VLM）和大语言模型（LLM）等算法能力，构建了一套通用的视频理解方案。该方案支持视频描述、结构解析、标签分类、问答场景、内容挖掘、视频检索、分析场景、营销应用场景、学习示例以及游戏赛事场景等十大常见应用，并内置多个对应的子任务模板。用户可以参照这些模板进行修改和调试，以适配或自定义其业务场景。总体而言，基于基础模型，系统支持更细粒度的理解和更复杂的任务；在功能上，支持一次VL视觉理解即可复用并完成多个下游子任务。

## **常见问题**

### **1、支持处理**哪些视频**？**

支持中英文视频，以及`.flv`、`.ts`、`.avi`、`.wmv`、`.mpg`、`.mkv`、`.mov`、`.mp4`格式，视频大小需小于200 MB，分辨率小于1080P，时长不超过10分钟。

### **2、如果有大量视频需要处理，怎样提高视频处理效率？**

视频处理API采取SSE协议，实时的流式处理方式，如果有大量视频需要处理，可以通过并行调用API的方式提高视频的处理效率，建议并发度控制在20 QPM以内，如果您需要更高并发可以联系我们处理。

### **3、视频处理中哪些步骤会消耗Token？怎样操作可以减少Token的消耗？**

-   视频处理大致流程分为：视频ASR提取、切片、抽帧、视觉语言分析（消耗VL大模型Token），文本加工（消耗文本大模型Token）。
    
-   其中视频时长增长对文本加工Token消耗影响不大，视频越长、抽帧间隔越短，视觉语言分析Token消耗基本呈线性增长。
    
-   如果您对Token消耗有要求，可以考虑通过增加抽帧间隔（snapshotInterval\[1~10\]）来降低Token消耗。但是抽帧间隔越长，视频理解的细腻程度会有所损耗。
    

### **4、离线调用怎么增加并发配额？**

您可以通过[视频理解-获取配置](https://help.aliyun.com/zh/model-studio/api-quanmiaolightapp-2024-08-01-getvideoanalysisconfig)、[视频理解-更新配置](https://help.aliyun.com/zh/model-studio/api-quanmiaolightapp-2024-08-01-updatevideoanalysisconfig)接口查看和扩缩并发。

### **5、**需要处理大量视频时，有什么方式可以降低费用**？**

在保证效果的前提下可尝试以下方式：

1.  VL和LLM用小尺寸模型。
    
2.  自定义抽帧配置（调整抽帧间隔，以及图片像素，将尺寸调小）。
    
3.  尝试关闭ASR（如果视频有字幕可以关闭）。
    
4.  将并发控制在2以内（2并发不收并发费用）。
    

### **6、视频中的声音和画面是怎么对应到同一段时间戳的？**

在视频理解过程中，系统通过以下方式将声音和画面对应到同一段时间戳：

-   自动语音识别（ASR）是结合视频画面进行的，画面和语音在解析时已经完成了切分。
    
-   视觉语言模型（VLM）在理解画面时，会对不同片段的画面记录对应的时间戳。
    
-   系统将VLM记录的画面时间戳与ASR语音识别的时间戳进行汇总对齐，从而实现声音和画面在同一时间段内的精准对应。
    

### **7、视频理解与Qwen3.6-thinking对比？**

**对比方面**

**视频理解与基模（qwen3.6-thinking）对比**

技术表现

-   视频理解：ASR+抽帧视觉分析；
    
-   基模：抽帧后基于视觉分析。
    

核心能力差异

①多语言：视频理解支持10+种语言的处理；基模不支持。

②角色识别：视频理解支持16个自定义角色人脸+ 公众人物 +人物聚合；基模仅支持公众人物识别。

③垂直模板：视频理解内置11个应用场景，29个prompt模板示例；基模不支持。

性能表现

①视频处理速度：处理1分钟长度的视频，基模处理速度更快。（视频理解需要3.6min，基模需要1.1min）。

②Token消耗：处理1分钟长度的视频，视频理解消耗的Token更少。（视频理解需要83k Tokens，基模需要285k Tokens）。

使用限制

①视频格式支持：视频理解支持 flv、ts、avi、wmv、mpg、mkv、mov、mp4；基模支持mp4、avi、mkv、mov、flv、wmv。

②视频时长限制：视频理解在线处理最长60 分钟，异步处理可扩展至4小时；基模支持 2秒至 2小时。

③视频数量限制：视频理解默认支持3 个，可根据需求扩展；基模最多支持 64 个视频。

④视频大小限制：视频理解默认支持1G，最高可扩展至4G；基模单个视频不超过 2G。

使用成本

相较于 VL+ PE 模式，视频理解消耗的 Token更少，且每 Token 的计费标准与基模保持一致，整体更具性价比。

能力效果

-   视频理解。
    

①支持音频理解，可根据音频、抽帧图片以及可能存在的角色信息和用户补充内容来理解；

②能捕捉的信息更多更细腻，通过VL和LLM串联的方式，可处理更复杂的任务；

③支持客户上传话者分离的srt、txt文件作为视频理解的补充。

-   基模。
    

①由于上下文的限制，无法处理复杂任务。
