# Prompt 工程

Prompt 工程是指通过系统化的方法设计、优化和管理提示词（Prompt），以引导大语言模型产生高质量、符合预期的输出。在百炼平台中，Prompt 工程贯穿应用构建、模型调用和效果评测的全流程，是充分发挥大模型能力的关键实践。

## 核心框架

百炼平台内置三种 Prompt 工程框架，适用于不同复杂度的任务：

| 框架 | 组成要素 | 适用场景 |
| --- | --- | --- |
| **ICIO** | 指令 + 背景信息 + 补充数据 + 输出格式 | 简单明确的任务（数据分析、内容生成、文本摘要） |
| **CRISPE** | 角色与能力 + 背景信息 + 任务 + 输出风格 + 输出范围 | 需要 AI 扮演特定角色的交互（客服、创意写作） |
| **RASCEF** | 角色 + 行动 + 步骤 + 上下文 + 示例 + 格式 | 多步骤复杂业务流程（项目规划、战略分析） |

在控制台创建自定义 Prompt 模板时，可选择"基于 Prompt 工程创建"模式，直接按上述框架进行结构化设计。

## 设计原则

### 文生文场景

- **清晰具体**：任务描述越清晰、无歧义，模型表现越能符合预期。
- **结构化组织**：按"背景 + 目的 + 风格 + 语气 + 受众 + 输出"六要素组织 Prompt。
- **Few-shot 示例**：提供少量输入输出示例，引导模型理解期望的格式和风格。
- **链式思考（CoT）**：引导模型逐步推理，在复杂逻辑任务中表现更优。

### 文生图场景

- **基础公式**：`主体 + 场景 + 风格`，适合快速上手。
- **进阶公式**：`主体描述 + 场景描述 + 定义风格 + 镜头语言 + 氛围词 + 细节修饰`，提升画面质感。
- 支持 `prompt`（正向提示词）与 `negative_prompt`（反向提示词）双向控制。

### 文生视频场景

- **基础公式**：`主体 + 场景 + 运动`。
- **进阶公式**：`主体描述 + 场景描述 + 运动描述 + 美学控制 + 风格化`。
- 图生视频场景因图像已确定主体和风格，Prompt 聚焦于 `运动 + 运镜`。

## 平台工具链

百炼围绕 Prompt 工程提供了完整的工具支持：

### Prompt 模板

将固定结构与动态变量分离，创建可复用的 Prompt 模板。模板分为预置模板（开箱即用）和自定义模板（适配复杂业务），可通过控制台、API 或 SDK 调用。变量最大支持 6144 个字符。

### 自动优化

大模型自动对原始 Prompt 进行结构重组、角色引导、指令增强和安全边界注入。该功能不计费，用户数据不会被存储或用于模型训练。

### 反馈优化

基于用户提供的样例数据和评测数据，通过多轮自动化评估、反思和优化生成更精准的 Prompt。建议提供 5~10 条样例数据和至少 20 条评测数据。

## 与应用评测的关系

Prompt 工程的效果需要通过系统化评测来验证。百炼的应用评测体系支持：

- 使用 LLM 评估器对 Prompt 产生的输出进行语义评分。
- 调整 Prompt 后触发评测，对比优化前后效果。
- 评测报告中的归因分析可定位问题是否出在 Prompt 层面，并给出针对性调优建议。

## 最佳实践

1. **从框架开始**：根据任务复杂度选择 ICIO / CRISPE / RASCEF 框架，避免无结构的自由编写。
2. **先自动优化再微调**：利用平台的自动优化功能快速提升基础质量，再根据业务需求手动调整。
3. **用数据驱动优化**：准备代表性样例，使用反馈优化功能迭代 Prompt。
4. **逻辑与内容分离**：通过 API 管理 Prompt，实现无需重新部署即可更新 Prompt 内容。
5. **持续评测**：每次修改 Prompt 后运行评测，确保变更带来正向改进。
