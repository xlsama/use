# 检索增强生成

检索增强生成（Retrieval-Augmented Generation，RAG）是一种在大模型生成回答前，先从外部知识库中召回与用户查询语义相关的文本切片，再将这些切片作为上下文注入提示词的技术，用于为模型补充私有数据和最新信息，提升特定领域问答的准确性与可信度。

## 核心原理

百炼平台的知识库能力即基于 RAG 实现，效果由三个阶段决定：

1. **建立索引**：将文件解析、切分为文本切片，再通过向量模型转换为向量，以键值对形式存入向量数据库。
2. **检索召回**：根据用户查询（Query）从向量存储中匹配并召回相关文本切片，基于语义相似度而非关键词匹配。
3. **生成答案**：大模型根据召回的文本切片和用户查询生成最终答案。

检索链路为：Query 向量化 → 初步向量检索 TopK + 初步关键词检索 TopK → Rerank 重排序 → 相似度阈值过滤 → 最终返回 TopK 切片。

## 在百炼平台的使用场景

### 智能体应用中的 RAG

在应用管理页面进入智能体应用 → 配置，点击文档知识库右侧 + 添加知识库。可配置：

- **相似度阈值**：仅语义相似度高于此阈值的切片被召回，过高会丢弃相关切片。
- **权重**：多个知识库同类型时，按权重对召回切片加权重排（仅在同类知识库间生效）。

调用方式建议设置为"必定调用"，确保 AI 能准确回答专业领域问题。

### 工作流应用中的 RAG

拖入"知识库"节点连接在"开始"节点之后，配置输入变量（值选内置变量 `query`）、选择知识库（固定或动态引入 `CodeList`）、可选设置 TopK。再拖入"大模型"节点，在提示词中通过 `/` 插入 `result` 变量引用检索结果。

### 外部应用（API）中的 RAG

通过阿里云百炼 SDK 调用知识库检索能力，仅适用于文档搜索类知识库。

### 通过开发框架集成 RAG

- **LlamaIndex（Python）**：使用 `DashScopeCloudIndex`、`DashScopeParse`、`DashScopeRerank` 等组件构建 RAG 应用，支持配置 `similarity_top_k`、`similarity_cutoff`、Rerank `top_n` 等参数。默认使用智能文档切分和官方向量模型，不支持自定义切分或嵌入模型。
- **Spring AI Alibaba（Java）**：通过 `DashScopeDocumentRetriever` 检索百炼云端知识库中的文本切片，结合 `DocumentRetrievalAdvisor` 将检索结果注入提示词，再用 `ChatClient` 调用大模型（默认 `qwen-max`）生成回答。

### 本地 RAG 应用

检索在本地执行（支持自定义切分策略和本地嵌入模型如 GTE 系列，避免大文件上传问题），生成则调用云端通义千问 API（qwen-max / qwen-plus / qwen-turbo），无需本地 GPU。可调参数包括模型温度、最大回复长度、召回片段数、相似度阈值等。环境要求 Python 3.8 - 3.12。

## 关键参数与配置

### 索引阶段

| 配置项 | 说明 | 备注 |
| --- | --- | --- |
| 切片方式 | 推荐**智能切分**（系统默认，效果最佳）；按长度切分需设重叠字符数（建议为最大分段长度的 10%–25%） | 创建后无法更改 |
| Meta 信息抽取 | 以 key-value 形式附加元数据到切片，检索前可结构化过滤；支持常量、变量（`file_name`/`cat_name`）、大模型、正则、关键词搜索 | 仅创建时可配置 |
| 解析方式 | 电子文档解析、文档智能解析、大模型文档解析、Qwen VL 解析、音视频解析 | 视数据类型选择 |
| Excel 表头拼装 | 将首行拼接到每个切片，避免表头被误识别为数据行 | — |

### 检索阶段

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| `similarity_top_k` | 检索返回的最高相似度结果数 | 5 |
| `similarity_cutoff` | 最低相似度过滤阈值 | 0.4 |
| `top_n`（Rerank） | 重排后返回的结果数 | 1 |
| `model_name` | 生成回答使用的大模型 | `qwen-max` |
| Rerank 模型 | 重排序模型 | `gte-rerank` |

### 多轮对话改写

根据历史会话自动补全用户查询，仅创建知识库时可开启并绑定到该知识库，未开启则后续无法再启用。

## 数据来源

知识库数据来源支持本地上传或从阿里云 OSS / 数据连接器导入。数据连接器提供文件、表格（平台托管）以及 MySQL、PostgreSQL、PolarDB-X 2.0、语雀、OSS（流处理）等多种类型，覆盖非结构化文档与结构化数据。单文档最大 100MB 或 1000 页，支持 pdf、doc、docx、txt、md 等格式。

## RAG 效果优化

当出现召回不完整或内容不准确时，按"建立评估基线 → 诊断 → 改进"三步迭代：

1. 借助自动评测创建至少 100 组问题的评测集，覆盖事实型 / 比较型 / 教程型 / 分析型用例，记录基线。
2. 针对失败用例（大模型打分 < 4）诊断问题来源。
3. 改进手段包括：补充相关知识、优化源文件排版（标题层次分明、移除水印、避免复杂合并单元格、优先 Markdown）、提示词语言与源文件语言一致、统一实体表述、启用多轮对话改写等。

## 规格

- **标准版**：0.03 元/小时，1 QPS（固定），平台存储 ≤ 100 GB，适合个人 / 小规模 / PoC。
- **旗舰版**：0.2 元/RCU/小时，50–10,000 QPS（可调，1–200 RCU），平台存储 ≤ 9,999 GB，适合高并发 / 生产级。

知识库类型与切片策略在创建后不可更改，metadata 抽取也只能在创建时配置。

## 支持的模型

预置模型与自定义模型均支持知识库，包括千问系列（QwQ / Long / Max / Plus / Turbo / Coder / Deep-Research）、千问 VL（Max / Plus / Flash / OCR）、千问开源版（Qwen3、Qwen2.5、Qwen2 等），以及 DeepSeek-R1、DeepSeek-V3.1、abab6.5s、Llama3.1、Yi-Large 等第三方模型。具体以应用管理页面创建应用时实际可选的模型为准。
