# knowledge base

阿里云百炼知识库（RAG）为大模型补充私有数据和最新信息，通过语义检索在生成回答前召回相关文本切片，从而提升特定领域问答的准确性。本文汇总了知识库的使用方式、效果优化、日志监控、配额限制、计费规则与 API 集成，供开发者快速落地与排障。详见 [知识库](../../raw/application-user-guide/knowledge-base/rag-knowledge-base.md)。

## 工作原理

知识库基于 RAG（[检索增强生成](../concepts/rag.md)）技术，效果由三个核心阶段决定：

1. **建立索引**：知识库将文件解析、切分为文本切片，再通过向量模型转换为向量，以键值对形式存入向量数据库。
2. **检索召回**：根据用户查询（Query），从向量存储中匹配并召回相关文本切片（语义相似度，而非关键词匹配）。
3. **生成答案**：大模型根据召回的文本切片和用户查询，生成最终答案。

检索链路为：Query 向量化 → 初步向量检索 TopK + 初步关键词检索 TopK → Rerank 重排序 → 相似度阈值过滤 → 最终返回 TopK 切片。详见 [RAG效果优化](../../raw/application-user-guide/knowledge-base/rag-optimization.md)。

## 支持的模型

预置模型（阿里云百炼标准模型）与自定义模型（基于以下模型调优后的模型）均支持知识库：

- 千问系列：QwQ / Long / Max / Plus / Turbo / Coder / Deep-Research
- 千问 VL：Max / Plus / Flash / OCR
- 千问开源版：Qwen3、Qwen2.5、Qwen2 等
- 第三方文本生成模型：DeepSeek-R1、DeepSeek-V3.1、abab6.5s、Llama3.1、Yi-Large 等

> **注意**：上述列表随时可能更新，请以在应用管理页面创建应用时实际可选的模型为准。

## 创建知识库

### 规格

创建时先选择规格，再按"填写基础信息 → 配置数据来源 → 设置索引参数"三步完成：

- **标准版**：0.03 元/小时，1 QPS（固定），平台存储 ≤ 100 GB，适合个人 / 小规模 / PoC。
- **旗舰版**：0.2 元/RCU/小时，50–10,000 QPS（可调，1–200 RCU），平台存储 ≤ 9,999 GB，适合高并发 / 生产级。

> **注意**：知识库类型与切片策略在创建后**不可更改**；metadata 抽取也只能在创建时配置，创建后无法再启用。

### 知识库类型

单一知识库不支持同时选择多个类型。文档搜索类还需选择使用场景：

- **基础文档问答**：纯文本文档的语义检索。
- **图文并茂回复**：返回图文混排内容。
- **视觉理解（富文本文档）**：使用多模态向量模型 `qwen3-vl-embedding` 对 PDF / 图片做视觉级理解，保留版面信息，支持文字 / 图片 / 图文组合三种命中模式。选择后向量模型自动切换且不可改。
- **极速问答**：针对低延迟优化，适合 FAQ、参数表等高度结构化文档，仅支持文本查询。

其他类型：数据查询类（结构化）、图片问答类、音视频搜索类。

### 数据来源与解析方式

数据来源支持本地上传或从阿里云 OSS / 数据连接器导入。文档搜索类知识库提供以下解析方式（不确定时保持默认）：

- **电子文档解析**：不支持插图与图表；最快，10–20 页纯文本文档数秒至 1 分钟。
- **文档智能解析**：对插图做文本识别 + 摘要；较快，10–20 页含插图 1–5 分钟。
- **大模型文档解析**：用千问 VL 深度理解插图 / 图表；2–10 分钟（10–20 页含图表）。
- **Qwen VL 解析**：专用于图片文件，单张数秒至 1 分钟。
- **音视频解析**：语音识别 + 视频帧提取 + 可选剧情解析；30 分钟音频约 5–15 分钟，1 小时视频约 15–60 分钟。剧情解析开启后耗时显著增加。

### 索引配置

关键配置项（影响检索效果，创建后多不可改）：

- **Meta 信息抽取**：以 key-value 形式把元数据附加到文本切片，检索前可据此结构化过滤。取值方法支持常量、变量（`file_name` / `cat_name`）、大模型（实体描述）、正则、关键词搜索。可分别控制是否参与检索、是否参与模型回复。**知识库创建后无法再配置**。
- **Excel 文件表头拼装**：将 XLSX/XLS 首行拼接到每个切片，避免表头被误识别为数据行。
- **切片方式**：推荐 **智能切分**（系统默认，经评估效果最佳，基于语义相关性自适应选择切片点）；另有按长度切分（需设重叠字符数，建议为最大分段长度的 10%–25%）。**创建后无法更改切分 chunk 策略**。

## 集成方式

知识库创建后可关联到同一[业务空间](../concepts/workspace.md)下的应用，或通过 SDK 对接外部应用。

### 智能体应用

在应用管理页面进入目标智能体应用 → 配置，点击文档知识库右侧 + 添加知识库。可配置：

- **相似度阈值**：仅语义相似度高于此阈值的切片被召回。过高会丢弃相关切片。
- **权重**：多个知识库同类型时，按权重对召回切片加权重排。**仅在同类知识库间生效**，文档搜索类权重不影响数据查询类召回顺序。

### 工作流应用

拖入"知识库"节点连接在"开始"节点之后，配置输入变量（值选内置变量 `query`）、选择知识库（固定或动态引入 `CodeList`）、可选设置 TopK。再拖入"大模型"节点，在提示词中通过 `/` 插入 `result` 变量引用检索结果。

### 外部应用（API）

通过阿里云百炼 SDK 调用知识库检索能力，具体步骤见 [知识库API指南](../../raw/application-user-guide/knowledge-base/rag-knowledge-base-api-guide.md)。仅适用于文档搜索类知识库。

## RAG 效果优化

当出现召回不完整或内容不准确时，按"建立评估基线 → 诊断 → 改进"三步迭代。建议先借助自动评测创建至少 100 组问题的评测集，覆盖事实型 / 比较型 / 教程型 / 分析型用例，记录基线后针对失败用例（大模型打分 < 4）改进。

### 检索无效：没有找到相关知识

- 补充相关知识到知识库。
- 优化源文件排版：标题层次分明、移除水印、避免复杂合并单元格表格、优先 Markdown（PDF/DOCX 建议先转 Markdown）。
- 提示词语言与源文件语言一致。
- 统一实体表述（如"ML" / "Machine Learning" / "机器学习"统一为"机器学习"）。
- 启用"多轮对话改写"：根据历史会话自动补全用户查询。**仅创建知识库时可开启，绑定到该知识库**，未开启则后续无法再启用。

### 检索无效：召回知识不相关

- **标签过滤**：为文件添加标签，检索前先按标签筛选文件（控制台或 `tags` 请求参数）。
- **提取元数据**：将结构化属性嵌入文本切片，检索前用 metadata 做结构化搜索精准定位目标文件。例如把产品名称作为元数据后，目标切片相似度可从 37% 提升至 65%。

### 切片不完整

切片过短（语义缺失）、过长（主题干扰）、语义截断都会降低效果。推荐使用 **智能切分**；切片后人工检查并编辑异常切片（保存后新内容生效，但源文件未修改，重新导入仍需检查）。

### 重排不佳

- 调整 **相似度阈值**：通过命中测试反复试验，过高（如 0.60）会丢弃所有相关切片。
- 调整 **召回片段数（K 值，1–20）**：对"列举 / 总结 / 比较"类问题适当增大。推荐选 **按拼装长度**，在不超过大模型最大输入长度的前提下尽量多召回。K 过大可能导致切片被截断反而影响效果。

### 模型理解有误

- **更换生成模型**：简单查询用 Flash / Turbo；复杂推理用 Max / QwQ；查阅大量文档用 Long / Plus；特定领域（如法律）用法睿。
- **优化提示词模板**：限定输出内容（如"信息不足时明确告知无法回答，切勿编造"）；用 Few-Shot 提供输出示例；用分隔标记把指令与 `${documents}` 变量分开，且 **`${documents}` 在模板中只出现一次**。

## 日志与监控

知识库所有检索调用投递到日志服务（SLS），支持调用审计、问题排查、用量统计与告警。开通后自动创建固定 Project / LogStore，按 SLS 实际产生费用计费。详见 [知识库日志与监控](../../raw/application-user-guide/knowledge-base/rag-knowledge-base-log-monitoring.md)。

### 开通与查看

在知识库列表页右上角点击"监控配置"：首次使用需授权角色 `AliyunServiceRoleForSFMAccessSLS`、开通 SLS、创建 LogStore（默认开启"检索日志"开关）。检索日志投递延迟通常为秒级。关闭"检索日志"开关只停止新日志投递，历史日志仍按 SLS 默认配置保留与计费；彻底停止计费需到 SLS 删除对应 LogStore。

### 关键字段

| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `request_id` | string | 单次请求唯一 ID |
| `pipeline_id` | string | 知识库 ID，可按知识库聚合 |
| `workspace_id` | string | [业务空间](../concepts/workspace.md) ID（格式 `llm-xxxx`） |
| `latency` | long | 请求耗时（毫秒） |
| `response_status_code` | long | HTTP 状态码 |
| `response_code` | string | 业务码，成功为 `Success`，失败为点分错误码（如 `Index.IndexNotExist`） |
| `response_body` | json | 响应体，召回切片在 `data.nodes[]`，每节点含 `score` / `text` / `metadata` |

### 常用 SQL

按知识库统计用量：`* | select pipeline_id, count(*) as cnt group by pipeline_id order by cnt desc`。建议监控业务错误率（`response_code != Success`）与 HTTP 5xx 错误率，阈值基于历史基线设定。

## 配额与限制

详见 [知识库配额与限制](../../raw/application-user-guide/knowledge-base/rag-knowledge-base-specifications.md)。

### 用量配额

| 类别 | 上限 |
| --- | --- |
| 知识库数量 | RDS 数据源 100；其它数据源无限制 |
| 平台存储容量 | 旗舰版 9,999 GB；标准版 100 GB |
| 类目数量（每[业务空间](../concepts/workspace.md)） | 500 |
| 文件数量（每业务空间） | 100,000 |
| 单个知识库文件数 | 非结构化无硬性上限；结构化 1 篇（单个 Excel/CSV） |
| 数据表数量（每业务空间） | 1,000 |
| ADB-PG 单表行数 / 单行大小 | 10,000,000 行 / 100 KB |

### 文件上传

文档搜索类：pdf/docx/doc/wps/pptx/ppt 最大 150MB 且页数 ≤ 1,000；txt/markdown/html 最大 10MB；xlsx/xls 最大 10MB 且 ≤ 10 万行；图片最大 20MB（短边 > 15 像素，长边 < 8,192 像素，长短边比例 < 50）。音视频搜索类最大 512MB。单次导入文件数 50（API 建议不超过 10,000），单文件标签数 32。

### 切片与向量化

- 文本切片数量（单文件）无限制；单切片 Token 上限 6,000。
- 切片操作：文档搜索 / 数据查询 / 图片问答类支持编辑、新增、删除；音视频搜索类支持编辑、删除（**不支持新增**）。`UpdateChunk` 长度 10–6000 字符，`DeleteChunk` 单次最多 10 个。
- 向量模型：文档搜索 / 数据查询 / 音视频搜索类支持 `text-embedding-v4`、`text-embedding-v3`（均 512 维）；图片问答类仅 `multimodal-embedding-v1`（1024 维）。向量维度不支持更改。

### 检索

- 检索并发：旗舰版 50–10,000 QPS（1–200 RCU）；标准版 1 QPS（固定）。
- 单次查询可召回文本切片数：20。

## 计费说明

知识库自 2026 年 1 月 4 日起正式计费。总费用 = **规格费用** + **模型调用费用**。详见 [知识库计费说明](../../raw/application-user-guide/knowledge-base/billing-for-knowledge-base.md)。

### 计费方式

后付费（按量付费）与预付费资源包。开通服务免费，从成功创建知识库开始计费。扣费顺序：**免费额度 > 资源包 > 按量付费**，按小时出账。删除知识库可停止计费（数据永久清除，不可恢复）。

### 规格费用

- 标准版：0.03 元/知识库/小时。
- 旗舰版：0.2 元/RCU/小时，所需 RCU = 向上取整（峰值 QPS ÷ 50），如 80 QPS 需 2 RCU。
- 自购 ADB-PG 需另付其费用。
- 变配：按变更发生时间分段计费，变配操作本身不收费；同一知识库 1 自然日内最多变配 1 次。

### 免费额度与资源包

- **免费额度**：所有用户一次性 720 小时，仅抵扣标准版规格费用，不含模型调用费。老用户有效期截至 2026 年 2 月 3 日 23:59；新用户自开通起 30 天内有效，过期作废。多个知识库按数量扣减（如 4 个标准版同时运行每小时扣 4 小时）。
- **资源包**：购买后一年内有效，剩余时长过期作废。标准版从 1 个/月（720 个·小时，20 元）到 100 个/年（18,999 元）；旗舰版从 1 RCU/月（139 元）到 50 RCU/年（65,999 元）。

### 模型调用费用

按输入 Token 量计费，遵循模型广场对应模型价格，不包含在规格费用内。公式：`模型费用 = (输入 Token 总数 / 1000) × 模型单价`。

- **创建 / 更新**：按新增内容 Token 数计费，删除文件不产生费用。
- **检索**：Query 向量化按输入 Token 计费；Rerank 费用取决于**初步召回总切片数**（非最终返回数），可配置关闭。多知识库时 Token 消耗按知识库数量倍数增加（N 个则 × N）。
- **费用优化**：关闭排序（降精度）；或降低初步向量检索 TopK 与初步关键词检索 TopK（取值 10–100）。

### 计费示例（text-embedding-v4 与 qwen3-rerank，均 0.0005 元/千 Token）

- 创建：50,000 Token 文件 → 0.025 元。
- 检索（单库）：100 Token Query + 召回 150 切片（均 500 Token）→ 0.00005 + 0.0375 = 0.03755 元。
- 检索（4 库）：0.03755 × 4 = 0.1502 元。

### 欠费处理

欠费后知识库进入暂停服务状态并停止计费。平台存储：0–14 天内补缴自动恢复，第 15 天视为放弃服务并永久删除数据。自购 ADB-PG：0–7 天内补缴恢复，第 8 天清理实例并永久删除。

## API 集成

通过阿里云百炼 SDK（`alibabacloud_bailian20231229`）调用知识库相关接口，适用于文档搜索类知识库。

### 前置步骤

1. 子账号需获取 `AliyunBailianDataFullAccess` 策略并加入一个业务空间（主账号可操作所有业务空间下的知识库）。
2. 安装最新版百炼 SDK。
3. 配置环境变量：`ALIBABA_CLOUD_ACCESS_KEY_ID`、`ALIBABA_CLOUD_ACCESS_KEY_SECRET`、`WORKSPACE_ID`。

### 创建知识库流程

1. 初始化 Client（endpoint 如 `bailian.cn-beijing.aliyuncs.com`）。
2. 计算文件 MD5 与大小，调用 `ApplyFileUploadLease` 申请上传租约。
3. 用租约返回的预签名 URL 与 headers，HTTP PUT 上传文件。
4. 调用 `AddFile`（指定 `parser`，如 `DASHSCOPE_DOCMIND`）将文件添加到类目，得到 `file_id`。
5. 轮询 `DescribeFile`，状态从 `INIT` → `PARSING` → `PARSE_SUCCESS` 后继续。
6. 调用 `CreateIndex`（指定 `structure_type` / `source_type` / `sink_type`）初始化知识库，得到 `index_id`。
7. 调用 `SubmitIndexJob` 提交索引任务，得到 `job_id`。
8. 轮询 `GetIndexJobStatus`，`COMPLETED` 后知识库可用。

> SDK 不满足需求时可通过签名机制 HTTP 请求相关接口。检索接口为 `/api/v1/indices/rag/index/retrieve`，单次最多召回 20 个切片。
