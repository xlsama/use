# Token 计量与计费

Token 是百炼平台衡量模型输入与输出文本量的基本单位，也是大语言模型推理计费的核心依据。不同模态的模型使用不同的计量单位（文本按 Token、图像按张、视频按秒等），但文本 Token 是最普遍的计费方式。

## Token 的含义

在大语言模型中，Token 是文本被分词后的最小处理单元。一个 Token 大致对应 1-2 个汉字或 3-4 个英文字母。百炼平台 100 万 Token 上下文约相当于 70 万汉字或 10 本小说。模型在处理请求时会分别统计输入 Token（用户发送的 [prompt](../guides/prompt.md)、上下文等）和输出 Token（模型生成的回复内容），两者分别计费且单价通常不同。

## 计费模式

### 按量付费

最基础的计费方式，按实际消耗的输入/输出 Token 数量计费。不同模型的单价差异明显：

| 模型层级 | 代表模型 | 输入单价（每百万 Token） | 输出单价（每百万 Token） |
|---------|---------|----------------------|----------------------|
| 旗舰模型 | qwen3.7-max | 12 元 | 36 元 |
| 均衡模型 | qwen3.7-plus | 2 元 | 8 元 |
| 轻量模型 | qwen3.6-flash | 更低 | 更低 |

部分模型实行阶梯计费：单次请求的输入 Token 总量越大，单价越高。例如 qwen3-max 在 0-32K 范围为 2.5 元/百万 Token，32K-128K 范围升至 4 元/百万 Token。

### 批量推理

支持 Batch 调用的模型，批量推理价格为实时推理价格的 50%，适合非实时、大批量处理任务。批量推理与上下文缓存折扣不能同时生效。

### 模型训练

训练费用按训练 Token 计费，公式为：（训练数据 Token 总数 + 混合训练数据 Token 总数）x 循环次数 x 训练单价。大规模模型（如 72b）训练单价为 0.15 元/千 Token，小规模模型（如 0.6b）低至 0.003 元/千 Token。

## Token Plan 与 Coding Plan

百炼还提供订阅制方案，适合有稳定用量的团队或个人：

- **Token Plan 团队版**：按 Credits 计量（底层仍基于 Token 消耗），198-1,398 元/坐席/月，支持文本和图像生成模型，多租户隔离不排队。
- **Coding Plan**：面向个人开发者，按模型调用次数计量（Pro 套餐 200 元/月），仅支持文本生成模型。

两者使用专属 API Key 和 Base URL，与按量计费的 API Key 互不相通。

## 容量保障：TPM 预留

TPM（Tokens Per Minute）预留可为指定模型锁定独占吞吐量，按 kTPM 预付费。预留容量内的调用不额外收费，超出部分自动降级为按量计费。适合流量可预估且对限流敏感的业务场景。

## 成本优化

### 免费额度

首次开通百炼可获得各模型的新人免费额度（有效期 30-90 天），仅抵扣华北2（北京）地域的实时推理费用。建议开启"免费额度用完即停"功能，避免额度耗尽后自动按量扣费。

### 节省计划与资源包

- **AI 通用型节省计划**：承诺每月消费金额换取阶梯式折扣，最高 5.3 折，覆盖全部阿里直供模型。
- **资源包**：预购特定 Token 数量，抵扣超出免费额度后的实时推理用量。
- **抵扣顺序**：免费额度 > 资源包 > 其他模型节省计划 > AI 通用型节省计划 > 按量付费。

### 上下文缓存

支持上下文缓存的模型，缓存命中的输入 Token 可享受折扣（如 Qwen 系列按 20% 折算，deepseek-v4-pro 按 8% 折算），显著降低重复上下文的成本。

## 用量监控

百炼控制台提供按[业务空间](workspace.md)维度的 Token 用量统计，支持按模型类型、推理类型和时间范围筛选。核心监控指标包括：

- **TPM**（Tokens Per Minute）：每分钟 Token 吞吐量
- **RPM**（Requests Per Minute）：每分钟请求数
- **平均单次请求调用量**：用于成本分析

建议配置用量告警，及时发现异常消耗。

## 开发者实践建议

- **合理设置 `max_tokens`**：控制单次生成的输出上限，避免不必要的 Token 消耗。
- **按任务选模型**：简单任务用轻量模型（如 qwen3.6-flash），复杂任务用旗舰模型，避免过度使用高价模型。
- **优化 Prompt**：简洁清晰的 Prompt 可减少输入 Token，同时提升输出质量。
- **利用批量推理**：非实时任务使用 Batch 调用，费用降低 50%。
- **善用缓存**：对重复上下文场景启用上下文缓存，大幅节省输入 Token 成本。
