# 图像生成 vs 视频生成 vs 3D 生成

百炼平台提供图像生成、视频生成和 3D 模型生成三大多媒体内容创作 API。三者均采用[异步任务调用](../concepts/async-task.md)模式，但在输入输出格式、支持的模型阵容、适用场景和计费方式上存在显著差异。本文从开发者技术选型角度，对三类能力进行全面对比，帮助快速定位最适合业务场景的 API。

## 关键维度对比

| 维度 | 图像生成 | 视频生成 | 3D 生成 |
|------|---------|---------|---------|
| **输入格式** | 文本提示词、参考图片、涂鸦草图、人像照片等 | 文本提示词、首帧/尾帧图片、参考视频/音频 | 文本提示词、单张图片、多图（2-4 张，前/左/后/右视角） |
| **输出格式** | 图片（JPEG/PNG），支持多张批量输出 | 视频文件（MP4），时长 1-10 秒不等 | 3D 模型（GLB 格式），可含 PBR 材质和渲染预览图 |
| **核心模型系列** | 千问 Qwen-Image、万相 Wan/Wanx、Z-Image、可灵 Kling | 万相 Wan、HappyHorse、可灵 Kling、PixVerse、Vidu | Tripo（Tripo-H3.1、Tripo-P1.0） |
| **模型数量** | 丰富（20+ 模型覆盖文生图、图编辑、风格迁移、试衣等） | 丰富（6 大系列覆盖文生视频、图生视频、人像动画等） | 较少（2 个模型，聚焦 3D 生成） |
| **推荐模型** | `qwen-image-2.0-pro`（文生图）、`wan2.7-image-pro`（编辑） | `wan2.7-t2v`（文生视频）、`wan2.7-i2v`（图生视频） | `Tripo/Tripo-H3.1`（高精度）、`Tripo/Tripo-P1.0`（快速） |
| **API 端点** | 多个端点，按模型类型区分（如 `/text2image/`、`/multimodal-generation/`） | 主端点 `/video-generation/video-synthesis`，部分旧模型用 `/image2video/` | `/video-generation/3d-generation` |
| **调用模式** | 异步（POST 创建任务 + GET 轮询结果） | 异步（POST 创建任务 + GET 轮询结果） | 异步（POST 创建任务 + GET 轮询结果） |
| **典型任务耗时** | 秒级至十秒级 | 1-5 分钟 | 分钟级 |
| **分辨率/精度控制** | `size` 参数（如 `1024*1024`），Pro 支持 4K | `resolution` 参数（480P-1080P） | `geometry_quality`（标准/ultra）、`texture_quality`（标清/高清），最高 200 万面 |
| **支持地域** | 北京（全部）、新加坡/弗吉尼亚（部分模型） | 北京（全部）、新加坡/弗吉尼亚/法兰克福（部分模型） | 仅北京 |
| **SDK 支持** | HTTP、Python SDK、Java SDK | HTTP、Python SDK、Java SDK | HTTP、Python SDK、Java SDK |
| **结果有效期** | task_id 24 小时 | task_id 24 小时 | task_id 24 小时，下载链接仅 2 小时 |

## 功能覆盖对比

| 能力 | 图像生成 | 视频生成 | 3D 生成 |
|------|:-------:|:-------:|:-------:|
| 文本驱动生成 | 支持 | 支持 | 支持 |
| 图片驱动生成 | 支持（图编辑、风格迁移） | 支持（首帧/首尾帧生视频） | 支持（单图/多图生 3D） |
| 多模态参考输入 | 支持（多图参考） | 支持（图片+视频+音频） | 支持（多视角图片） |
| 内容编辑/重绘 | 支持（局部重绘、扩图、擦除） | 支持（指令编辑、视频迁移） | 不支持 |
| 人像/人物相关 | 支持（试衣、写真、风格重绘） | 支持（数字人、换人、动画） | 不支持 |
| PBR 材质输出 | 不适用 | 不适用 | 支持 |

## 适用场景建议

### 图像生成

适用于需要快速产出静态视觉内容的场景：

- 电商商品主图、海报、广告素材批量生成
- 图片编辑与后期处理（扩图、擦除水印、背景替换）
- 虚拟试衣、人像写真等互动体验
- 内容创作中的插图、概念设计快速迭代

图像生成模型数量最多、场景覆盖最广，且响应速度快，适合对实时性有要求的在线服务。

### 视频生成

适用于需要动态视觉内容的场景：

- 短视频、广告片的自动化生产
- 数字人播报、唱演视频制作
- 动画角色驱动和动作迁移
- 视频风格转换与创意后期

视频生成耗时较长但产出内容表现力最强，适合对内容丰富度和表现力有高要求的营销、娱乐场景。

### 3D 生成

适用于需要三维模型资产的场景：

- 游戏、AR/VR 中的 3D 资产快速原型制作
- 电商商品 3D 展示与互动浏览
- 工业设计、建筑可视化中的概念验证
- 3D 打印前的模型快速生成

3D 生成当前模型选择较少且仅限北京地域，但能直接输出可用的 GLB 格式模型，适合需要三维资产的垂直场景。

## 技术选型建议

1. **按输出需求选择**：需要静态图片选图像生成，需要动态内容选视频生成，需要三维模型选 3D 生成。
2. **按响应速度选择**：图像生成最快（秒级），3D 和视频生成均需分钟级等待。若对延迟敏感，优先考虑图像生成。
3. **按模型生态选择**：图像和视频生成拥有多个模型系列可选，便于横向对比效果；3D 生成目前仅 Tripo 一家，选择空间有限。
4. **按地域需求选择**：若需海外部署，图像和视频生成部分模型支持新加坡/弗吉尼亚；3D 生成仅限北京地域。
5. **注意结果时效**：三者的 task_id 均有 24 小时有效期，但 3D 生成的下载链接仅 2 小时有效，需在架构中考虑及时拉取和持久化存储。
