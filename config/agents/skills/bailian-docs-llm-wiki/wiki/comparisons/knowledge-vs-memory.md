# 知识库与长期记忆对比

百炼平台提供了两种为大模型补充上下文信息的机制：**知识库**（基于 [RAG 检索增强生成](../concepts/rag.md)）和**长期记忆（新）**。两者都能让模型在生成回答时获得额外信息，但在设计目标、数据来源、集成方式和适用场景上存在显著差异。本文从开发者技术选型的角度，对两者进行系统对比。

## 核心定位差异

- **知识库**面向**文档级知识管理**，将企业私有文档（PDF、Word、音视频等）切片后建立向量索引，通过语义检索为模型提供结构化的参考资料。核心价值是让模型"读懂企业文档"。
- **长期记忆（新）**面向**用户级对话记忆**，从多轮对话中自动提取关键信息并构建用户画像，在后续交互中基于语义相似度召回相关记忆。核心价值是让模型"记住用户偏好"。

## 关键维度对比

| 维度 | 知识库 | 长期记忆（新） |
|------|--------|---------------|
| **核心技术** | RAG（检索增强生成），向量化 + 语义检索 | 对话记忆提取 + 语义检索 |
| **数据来源** | 企业文档文件（PDF/DOCX/TXT/Markdown/音视频等） | 用户对话消息或自定义文本片段 |
| **数据粒度** | 文档切片（最大 6,000 Token/片） | 记忆片段（自定义内容最大 512 字符） |
| **数据写入方式** | 上传文件，系统自动切片和向量化 | 通过 API 传入对话记录或自定义内容，系统自动提取记忆 |
| **检索方式** | 向量检索 + 可选 Rerank 排序 | 语义相似度检索，支持 Rerank、意图判别、Query 重写 |
| **检索参数** | 相似度阈值 + TopK（最多 20 个切片） | top_k（1~100）+ min_score（0~1）+ 多种增强选项 |
| **API 端点** | 百炼 SDK 知识库检索接口 | RESTful API（Base URL: `https://dashscope.aliyuncs.com/api/v2/apps/memory/`） |
| **认证方式** | AK/SK（AccessKey + Secret） | API Key（Bearer Token） |
| **集成方式** | 智能体应用 / 工作流节点 / SDK API | RESTful API / Python SDK（agentscope-runtime） |
| **数据组织** | 知识库 > 类目 > 文件 > 切片，支持标签和元数据 | 记忆库 > 用户 > 记忆片段，支持画像模板 |
| **用户维度** | 无内置用户隔离，按知识库整体管理 | 按 user_id 天然隔离，每个用户独立记忆空间 |
| **数据更新** | 重新上传文件触发重新切片和向量化 | 通过 API 增删改记忆片段（增量更新） |
| **存储上限** | 标准版 100 GB / 旗舰版 9,999 GB | 无明确存储上限（记忆片段无失效日期） |
| **并发限制** | 标准版 1 QPS / 旗舰版 50~10,000 QPS | 全接口 3,000 QPM，add 120 QPM，search 300 QPM |
| **计费方式** | 规格费（按小时）+ 模型调用费（向量化 + 检索） | 按 API 调用计费 |
| **费用起点** | 标准版 0.03 元/知识库/小时，旗舰版 0.2 元/RCU/小时 | 以 API 调用量计费 |
| **支持的知识类型** | 文档搜索、数据查询、图片问答、音视频搜索 | 对话记忆、用户画像 |

## 功能特性对比

| 特性 | 知识库 | 长期记忆（新） |
|------|--------|---------------|
| 文档文件上传 | 支持（PDF/DOCX/音视频等多格式） | 不支持 |
| 自动切片 | 支持（智能切分 / 按长度切分） | 不适用（系统自动提取记忆片段） |
| 元数据抽取 | 支持（常量/变量/大模型/正则/关键词） | 支持 meta_data 自定义字段 |
| 用户画像 | 不支持 | 支持（画像模板 + 自动提取） |
| 多模态支持 | 支持（图片/音视频向量化） | 不支持 |
| 可视化管理 | 控制台 UI（知识库管理页面） | 纯 API 操作 |
| 日志与监控 | 支持（日志投递至 SLS） | 无内置日志服务 |

## 适用场景建议

### 选择知识库的场景

- **企业知识问答**：需要基于大量内部文档（产品手册、技术文档、FAQ）构建问答系统
- **多模态检索**：需要对图片、音视频等非文本内容进行语义搜索
- **结构化数据查询**：需要对 Excel/CSV 数据进行自然语言查询
- **高并发检索**：业务场景对检索 QPS 有较高要求（旗舰版支持万级 QPS）
- **合规审计**：需要检索日志投递和调用审计能力

### 选择长期记忆的场景

- **个性化对话助手**：需要记住用户偏好、历史交互，提供个性化回复
- **用户画像构建**：需要从对话中自动提取用户特征，构建结构化画像
- **多轮对话增强**：跨会话保持上下文连贯性，避免用户重复提供信息
- **轻量级集成**：只需 RESTful API 即可完成集成，无需管理文档文件

### 组合使用

两者并非互斥，在实际业务中可以组合使用：知识库提供领域知识检索能力，长期记忆提供用户个性化上下文。例如，一个客服智能体可以同时接入知识库（检索产品文档）和长期记忆（记住用户历史问题和偏好），从而在提供准确答案的同时实现个性化服务体验。

## 技术选型决策参考

| 决策因素 | 推荐方案 |
|----------|----------|
| 需要基于企业文档回答问题 | 知识库 |
| 需要记住用户偏好和历史 | 长期记忆 |
| 数据以文件形式存在 | 知识库 |
| 数据来自用户对话 | 长期记忆 |
| 需要多模态检索 | 知识库 |
| 需要用户画像能力 | 长期记忆 |
| 需要控制台可视化管理 | 知识库 |
| 需要纯 API 轻量集成 | 长期记忆 |
| 需要高并发检索（>300 QPM） | 知识库（旗舰版） |
| 同时需要知识和个性化 | 两者组合使用 |
