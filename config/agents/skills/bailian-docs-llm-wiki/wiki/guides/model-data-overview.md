# model data overview

百炼平台的数据管理功能为模型调优和评测提供统一的数据集管理能力，覆盖训练集与评测集的创建、格式约束、版本管理，以及对训练集进行清洗与增强的数据流编排。本页汇总两类数据集的格式规范、关键参数与使用注意事项，帮助开发者在正式训练前准备好高质量数据源。

## 数据集类型

数据集在 [数据管理](https://bailian.console.aliyun.com/#/efm/model_data) 页面下统一管理，分为两类：

- **训练集**：用于模型调优（SFT、DPO、CPT、图生视频等），通过在特定任务数据上有监督训练提升模型在特定任务上的表现。详见 [训练集与评测集](../../raw/model-user-guide/model-data-overview/training-set-and-evaluation-set.md)。
- **评测集**：用于评估调优后模型的泛化能力，即在未见过数据上的表现。目前仅支持**文本生成**评测集。

> **注意**：训练集与评测集相关文档仅适用于华北2（北京）地域；数据清洗或增强文档仅适用于中国大陆版（北京地域）。开发者在跨地域使用前需确认地域一致性。

### 训练集规模要求

| 调优方式 | 最少数据量 |
| --- | --- |
| CPT | 一千万 Token 优质预训练数据 |
| SFT | 上千条优质微调数据（构建有效 SFT 训练集通常需要 1000+ 样本） |
| DPO | 上百条人类偏好数据 |

## 文本生成与多模态理解训练集

### SFT 训练集（ChatML）

采用 ChatML 格式，支持多轮对话和 system / user / assistant 多种角色。单行训练数据为 JSON 对象，`messages` 数组中所有 assistant 行都会被训练。

- **不支持** OpenAI 的 `name`、`weight` 参数。
- 单条训练数据的所有 assistant 行支持 `loss_weight` 参数（范围 `0.0 ~ 1.0`，数值越大重要性越高），属邀测参数，需联系商务经理开通。
- 支持 `.jsonl`、`.xls`、`.xlsx` 样例格式（xls/xlsx 仅支持单轮对话）。

```
{"messages": [
  {"role": "system", "content": "系统输入1"},
  {"role": "user", "content": "用户输入1"},
  {"role": "assistant", "content": "期望的模型输出1"},
  {"role": "user", "content": "用户输入2"},
  {"role": "assistant", "content": "期望的模型输出2", "loss_weight": 1.0}
]}
```

### SFT 思考模型（thinking）

针对支持思考模式（`<think>` 标签）的模型，训练数据需遵循以下规则：

- 思考标签 `<think>` 前后的若干个 `\n` 必须保留。
- 只能针对**最后**的 assistant 输出进行训练。
- 思考内容只能包含在最后一个 assistant 输出中，中间 assistant 输出不应添加 `<think>` 标签。
- 最后的 assistant 行支持 `loss_weight` 参数（同样为邀测参数）。

若训练样本中设置模型不输出 `<think>` 标签，训练完成后**不建议**再开启思考模式进行调用。

### SFT 视觉理解（千问 VL）

ChatML 格式，`content` 必须使用数组格式，例如 `[{"text":"..."}]`，不能使用字符串格式。支持图片与视频输入：

- 图片字段 `image`（必填）、`resized_width`、`resized_height`（可选）。
- 视频文件路径模式（仅 qwen3.5 及以后 VL 模型支持）：`video`（str）、`fps`、`video_start`、`video_end`、`resized_width`、`resized_height`。
- 图片帧列表模式（仅 qwen3.5 及以后 VL 模型支持）：`video`（List[str]）、`sample_fps`、`resized_width`、`resized_height`。

**训练物体定位建议**：
- Qwen2.5-VL：训练坐标相对于缩放后图像左上角的绝对值，单位为像素。
- Qwen3-VL：训练坐标为相对坐标，坐标值会缩放到 `[0, 999]` 范围内。

**压缩包要求**：
1. 格式为 ZIP，最大支持 2 GB，包内文件夹与文件名仅支持 ASCII 字符集中的字母、数字、下划线、连字符。
2. 训练文本数据固定为 `data.jsonl`，必须位于压缩包**根目录**下，压缩后打开 zip 应直接看到该文件。
3. 图片单张宽度和高度均不超过 1024px，单张不超过 10MB，支持 `.bmp`、`.jpeg/.jpg`、`.png`、`.tif/.tiff`、`.webp`。
4. 图片文件名在压缩包内必须全局唯一，即使分布在不同的文件夹中。

### DPO 数据集

DPO ChatML 格式，`messages` 内所有内容作为输入，`chosen` / `rejected` 分别表示赞同与反对的模型期望输出。针对深度思考内容需使用 `<think>` 标签包裹。`chosen` 模块支持 `loss_weight` 参数（邀测参数）。

```
{"messages": [...],
 "chosen": {"role": "assistant", "content": "赞同的模型期望输出3", "loss_weight": 1.0},
 "rejected": {"role": "assistant", "content": "反对的模型期望输出3"}}
```

### CPT 训练集

CPT 纯文本格式，单行结构为：

```
{"text":"文本内容"}
```

## 图生视频训练集

图生视频训练集分为**基于首帧**与**基于首尾帧**两类，均要求训练集必须提供，验证集可选。详见 [训练集与评测集](../../raw/model-user-guide/model-data-overview/training-set-and-evaluation-set.md)。

### 基于首帧

- 训练集包含：首帧图像、训练视频、标注文件 `data.jsonl`（最大 20MB）。
- 图像最大分辨率 4096x4096，支持 BMP、JPEG、PNG、WEBP；视频最大 4096x4096，支持 MP4、MOV。
- 验证集无需提供视频，训练任务在评估节点自动调用模型服务生成预览视频。

标注结构：

```
{"prompt": "...", "first_frame_path": "image_1.jpg", "video_path": "video_1.mp4"}
```

### 基于首尾帧

- 训练集包含：首帧图像、尾帧图像、训练视频、标注文件 `data.jsonl`。
- 压缩包内 `image/` 与 `video/` 子目录分别存放图像与视频。
- 验证集包含首尾帧图像与 `data.jsonl`，无需视频。

标注结构：

```
{"prompt": "...", "first_frame_path": "image/image_1_first.jpg", "last_frame_path": "image/image_1_last.jpg", "video_path": "video/video_1.mp4"}
```

## 文本生成评测集

单轮对话评测数据，采用 Excel 格式，每行包含 Prompt 与 Completion 两列。在模型评测中，参评模型基于评测集中每条 Prompt 进行推理，评分员或自动化评分系统参考 Completion 对推理结果评分。

## 数据清洗与增强

数据处理通过对训练集执行**数据清洗**与**数据增强**获得更高质量数据源。当前仅支持 [SFT-文本生成训练集](../../raw/model-user-guide/model-data-overview/data-processing.md)，不支持 SFT-图片理解训练集与 DPO-文本生成训练集。

> **注意**：若训练集数据不适合清洗与增强（如法律文件、医学记录、文学作品、方言汇总、用户评论、技术手册等），建议直接跳过数据处理步骤。阿里云百炼目前暂未提供可用的数据处理 API，操作均需在控制台完成。

### 清洗与增强能力

| 处理方式 | 适用场景 |
| --- | --- |
| 数据清洗 | 修正训练数据中的规范性、合规性、一致性及重复问题。目前已支持**特殊内容移除**、**敏感信息打码**等十种清洗操作 |
| 数据增强 | 增加训练数据的多样性和均衡性，或扩展数据规模（增广数据） |

> 推荐先清洗再增强：确保增强操作基于干净、高质量的数据集，保证模型调优数据源准确。

### 数据流编排

通过控制台 [数据管理](https://bailian.console.aliyun.com/?tab=model#/efm/model_data) 页面搭建自定义数据流，典型两步：

1. **创建数据流**：拖入开始/数据清洗/数据增强/结束节点并连接，发布数据流。
2. **创建数据流任务**：选择已发布数据流、数据来源（模型数据）与目标训练集，启动任务。

开始节点预设 `对话文本` 参数表示待处理训练集（不可更改）；数据清洗节点预设 `dataSetCount` 表示处理后训练集大小（不可更改）；结束节点输出会自动生成训练集新版本，新版本独立保存，不会覆盖原训练集。

### 节点说明

- **开始/结束节点**：每个数据流必须包含一个开始节点与一个结束节点。结束节点为最后节点，输出处理后的训练集，后面不可再添加节点。
- **条件判断节点**：支持且/或条件配置，多个条件自上而下顺序执行；提供"其他"分支用于无需条件判断的链路。
- **数据清洗节点**：选择清洗算子（如文章相似度去重、敏感词过滤、毒性消除等）与操作，算子按编排顺序执行。可通过拖拽调整算子顺序；建议每个节点只开启一个算子以便排查问题。
- **数据增强节点**：基于 `千问-Max` 大模型，通过 Few-Shot 策略生成新数据。场景选项包括：
  - 数据增强-通用：适用于增强文本问答大模型所需的 SFT 训练集，每次最多生成 2000 条样本。
  - 数据增强-文本分类 / 文本抽取 / 文本创作：在通用基础上经场景优化，对应特定任务增强效果优于通用模式。

### 数据增强关键参数

| 参数 | 说明 |
| --- | --- |
| 生成样本数 | 节点需要生成的数据量。原 10 条 + 生成 10 条 = 总计 20 条；通用模式单次最多 2000 条 |
| 指令生成依赖样本数 | 从增强前训练集中选出的种子数量，拼入 Prompt 提供给大模型；若种子加 Prompt 超过千问-Max 最大输入 Token，系统自动调整 |
| 过滤相似度阈值 | 过滤生成数据中的相似样本，保持默认即可 |
| Prompt 配置 | 定义增强任务的输入输出要求，支持 `few_shot_examples` 参数，系统提供默认模板 |

> 增强后输出数据中的 `foreignKey` 为系统后添加，用于标识文本信息，可直接用于模型调优，无需删除。

### 任务状态与管理

数据流任务处理状态包括：

- **处理中**：任务执行中，高峰时段需排队等待，期间无需介入。
- **已完成**：任务成功完成，可在执行过程查看处理结果与每个节点耗时。
- **处理失败**：任务执行失败，建议提交工单咨询具体原因。

任务执行期间暂不支持手动终止；任务完成后会为训练集生成新版本，建议检查清洗后的训练集以确保数据完整性与真实性未被破坏。

## 限制与注意事项

- 文档适用地域仅限华北2（北京）/中国大陆版（北京地域），跨地域使用前需确认。
- 数据处理暂未提供 API，仅支持控制台操作。
- 数据处理仅支持 SFT-文本生成训练集（ChatML 格式），不支持图片理解与 DPO 训练集。
- `loss_weight`、`name`、`weight` 等参数中，`name`/`weight` 不被支持，`loss_weight` 属邀测参数，需联系商务经理开通。
- 思考模型训练样本中，`<think>` 标签前后的 `\n` 必须保留，思考内容只能包含在最后一个 assistant 输出中。
- VL 训练集中图片文件名须在压缩包内全局唯一；视频文件路径模式与图片帧列表模式仅 qwen3.5 及以后 VL 模型支持。
- 图生视频训练集的 `data.jsonl` 最大 20MB，图像/视频最大分辨率 4096x4096。
- 数据增强-通用任务每次最多生成 2000 条样本，且仅支持 `few_shot_examples` 参数。
