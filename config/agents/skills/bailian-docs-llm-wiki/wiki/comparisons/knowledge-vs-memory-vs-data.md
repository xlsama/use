# 知识库 vs 记忆库 vs 数据连接

百炼平台提供了三种为大模型补充外部数据的机制：**知识库**、**记忆库**和**数据连接**。三者解决的核心问题各不相同——知识库侧重私有文档的语义[检索增强生成（RAG）](../concepts/rag.md)，记忆库侧重跨会话的用户级长期记忆，数据连接侧重对企业已有数据源的实时接入。开发者在技术选型时往往需要将它们组合使用，本文从关键维度对比三者差异，帮助快速判断适用场景。

## 关键维度对比

| 维度 | 知识库 | 记忆库 | 数据连接 |
|------|--------|--------|----------|
| **核心定位** | 基于 RAG 为大模型补充私有文档和最新信息 | 为智能体提供跨会话的长期记忆能力 | 统一管理外部数据源，实时访问企业数据 |
| **数据来源** | 用户上传的文档（PDF、Word、Markdown、音视频等） | 对话过程中自动提取的记忆片段或用户画像 | 外部数据库（MySQL/PostgreSQL/PolarDB-X）、语雀、OSS、文件、表格 |
| **数据存储** | 平台托管（标准版/旗舰版），向量化后持久存储 | 平台托管，按 user_id 隔离的记忆空间 | 平台托管（文件/表格）或数据保留在原数据源（流处理类型） |
| **检索方式** | 语义向量检索 + 关键词检索 + Rerank 排序 | 基于语义的记忆检索（SearchMemory） | 智能体自动调用连接器工具查询（如 SQL 查询、文件搜索） |
| **数据更新** | 手动上传/更新文档，支持 OSS/飞书/钉钉自动同步 | 每轮对话后调用 AddMemory 自动提取和更新 | 流处理类型实时访问源数据，文件/表格需手动或规则同步 |
| **API 端点** | 百炼 SDK 知识库检索 API | `https://dashscope.aliyuncs.com/api/v2/apps/memory/` 系列接口 | 通过智能体/工作流内置的连接器工具调用 |
| **集成方式** | 智能体应用、工作流节点、SDK API 调用 | API 调用（AddMemory/SearchMemory）、OpenClaw 插件 | 智能体技能、工作流数据连接器节点 |
| **支持的模型** | 千问全系列、DeepSeek 等第三方模型，支持自定义微调模型 | 不直接绑定推理模型，与任意应用/模型配合使用 | 不直接绑定推理模型，由智能体/工作流中的模型调用 |
| **计费方式** | 规格费（标准版 0.03 元/库/小时，旗舰版 0.2 元/RCU/小时）+ 向量模型调用费 | 限时免费 | 文件/表格连接器限时免费（平台存储 1 TB 额度），数据库连接器需开通相关云服务 |
| **并发/速率** | 标准版 1 QPS，旗舰版 50-10,000 QPS | AddMemory 120 QPM，SearchMemory 300 QPM，合计 3000 QPM | 取决于底层数据源和连接器类型 |
| **典型延迟** | 取决于知识库规格和检索参数 | SearchMemory 200-500ms，AddMemory 500-1000ms | 取决于数据源响应速度 |
| **存储限制** | 标准版 100 GB / 旗舰版 9,999 GB，单空间最多 100,000 文件 | 无明确存储上限（限时免费期间） | 文件连接器 100,000 文件 / 1 TB，数据库单表最大 10,000,000 行 |

## 适用场景建议

### 选择知识库

- 企业有大量私有文档（产品手册、技术文档、FAQ）需要被大模型引用。
- 需要对非结构化文档进行语义检索，让模型基于检索结果生成准确回答。
- 场景涉及图文并茂回复、富文本文档理解、音视频内容搜索等多模态需求。
- 对检索并发有较高要求（旗舰版支持最高 10,000 QPS）。

### 选择记忆库

- 智能体需要"记住"用户的偏好、习惯和历史交互信息，跨会话提供个性化服务。
- 需要从对话中自动提取关键事件，无需用户手动录入。
- 需要构建用户画像，按模板持久化存储用户属性（年龄、职业、偏好等）。
- 多个应用需要共享同一份用户记忆。

### 选择数据连接

- 企业数据存储在 MySQL、PostgreSQL 等关系型数据库中，需要在对话中实时查询。
- 需要访问语雀文档库、OSS 对象存储中的文件。
- 数据需要保留在原有存储系统中，不希望复制到百炼平台。
- 需要从飞书、钉钉等协作平台自动同步文档到百炼应用。

## 组合使用建议

三种机制并非互斥，实际项目中常组合使用以覆盖不同数据需求：

- **知识库 + 记忆库**：知识库提供专业知识检索，记忆库保留用户个性化信息，两者结合实现"既专业又贴心"的智能体。
- **知识库 + 数据连接**：知识库处理非结构化文档检索，数据连接处理结构化数据库实时查询，覆盖企业全类型数据。
- **三者结合**：在复杂企业场景中，同时启用知识库（文档 RAG）、记忆库（用户画像与偏好）、数据连接（业务数据库实时查询），构建全面的智能体数据层。
