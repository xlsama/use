# 应用评估与模型评估对比

百炼平台提供两套相互独立但互补的评估能力：**应用评估**针对已经搭建完成的智能体应用/工作流应用，验证端到端的输出质量；**模型评估**针对底层大语言模型本身，验证其推理能力。两者评估对象、数据组织、评分机制和计费方式均不同，开发者应根据技术选型阶段选择合适的评估路径。

## 关键维度对比

| 维度 | 应用评估 | 模型评估 |
|------|----------|----------|
| 评估对象 | 已发布且配置知识库的智能体应用、工作流应用 | 预置模型及基于其调优后的模型 |
| 评估目标 | 端到端应用输出质量（含检索、Prompt、模型协同） | 模型本身的推理能力 |
| 前置条件 | 应用已发布、开通应用观测、子账号具备评测权限 | 需有可评测的模型（基线评测仅支持调优后模型） |
| 评测模式 | 自动评测（单应用/多应用横向）、手动评测 | 自定义评测、基线评测 |
| 评测集来源 | 自动生成（基于知识库）、手动上传 xls/xlsx、从应用观测导入 | 评测数据集（Prompt+Completion）、推理结果集（含 Output）、基线预置数据集 |
| 评测集格式 | 对话分析 `.xls/.xlsx`、知识问答 `.jsonl`、新版智能体/工作流/自定义表结构 | Excel 格式（Prompt、Completion、可选 Output） |
| 评分方式 | 评估器（预置模板、LLM 评估器、Code 评估器）+ 标签人工标注 | 评测维度（大模型评估数值型/分类型、规则评估相似度/字符串匹配、人工评估分类型） |
| 评分核心变量 | 评估器参数映射评测集字段 | `${prompt}`、`${output}`、`${completion}` 三变量 |
| 支持模型 | 评测生成与执行仅支持 qwen-max、qwen-plus | 千问全系列（QwQ、Max、Plus、Turbo、Coder、Math、开源版）、通义法睿、第三方模型 |
| API/入口 | 控制台应用评估模块、应用观测 | 控制台模型评测模块、创建评测任务页面 |
| 评测报告 | 总正确率、BadCase、归因分析（模型理解/重排/检索/切片/未获取知识）、调优建议 | 综合得分、通过率、数据明细；基线评测额外提供分学科得分、能力雷达图 |
| 横向对比 | 多应用横向评测（最多 8 个） | 排行榜（多模型同一评测维度对比） |
| 计费方式 | 自动评测和 LLM 评估器按 Token 实际消耗计费；Code 评估器无额外 Token 费用 | 大模型评估维度产生 Token 费用；规则评估类型成本极低；人工评估成本最高 |
| 典型场景 | 应用上线前后质量回归、知识库/Prompt 调优闭环、多应用选型 | 模型选型、调优验证、能力量化、持续监控 |

## 适用场景建议

### 选择应用评估

- 已经在百炼搭建了智能体或工作流应用，需要验证端到端业务效果。
- 关注检索增强（RAG）链路质量，需要归因到切片、重排、检索等具体环节。
- 希望基于知识库自动生成评测集，快速建立持续评测闭环。
- 需要在多个应用版本或多应用之间做横向对比选型。

### 选择模型评估

- 处于模型选型阶段，尚未搭建应用，需要用业务测试题对比候选模型。
- 需要验证模型调优（微调）前后的能力提升幅度。
- 需要使用行业标准数据集（C-Eval、MMLU、ARC、GSM8K、HellaSwag、BBH）做通用能力基线测试。
- 需要量化的综合得分、通过率和能力雷达图用于团队汇报或管理层决策。

### 组合使用

技术选型阶段先用模型评估筛出合适模型，应用搭建完成后再用应用评估验证端到端效果。两者评测报告可共同构成从模型层到应用层的完整质量证据链。

## 技术选型参考

- **评估粒度**：仅关心模型推理能力选模型评估；关心应用整体表现选应用评估。
- **数据准备成本**：基线评测无需准备数据，适合快速验证；自定义评测和应用评测均需准备数据，但应用评估支持基于知识库自动生成，成本更低。
- **评分灵活性**：模型评估的 5 种评测维度偏标准化；应用评估的评估器（预置/LLM/Code）+ 标签组合更适合复杂业务规则。
- **归因深度**：应用评估提供 BadCase 归因和调优建议，适合迭代优化；模型评估提供能力雷达图和排行榜，适合横向对比。
- **费用控制**：优先使用规则评估、Code 评估器和基线评测等低/零 Token 方案，大模型评估和自动评测按需使用。
