# model monitoring

阿里云百炼平台提供模型监控与用量统计功能，帮助开发者实时掌握模型调用状况、Token 消耗趋势以及性能表现。通过监控看板、调用日志和主动告警机制，开发者可以及时发现异常并优化成本。

## 功能概览

百炼的模型监控体系包含两个核心模块：

- **模型用量**：按[业务空间](../concepts/workspace.md)维度统计各模型的调用量和 Token 消耗，支持按模型类型、推理类型和时间范围筛选，数据延迟约 1 小时。详见 [模型用量](../../raw/model-user-guide/model-monitoring/model-usage-statistics.md)。
- **模型监控**：提供调用记录查看、性能指标监控（Token 延时、调用时长、RPM、TPM、失败率）以及主动告警能力。详见 [模型监控](../../raw/model-user-guide/model-monitoring/model-telemetry.md)。

## 支持的模型

- **用量统计**：模型列表中的所有模型均支持查看用量，包括基于它们调优后的自定义模型。
- **普通监控**：支持模型列表中的所有模型及自定义模型。
- **高级监控**：支持北京、新加坡、弗吉尼亚地域下的所有模型。
- **告警功能**：支持北京、新加坡地域下的所有模型。
- **日志功能**：仅支持北京和新加坡地域下的部分模型（如 qwen3-max、qwen-plus、deepseek-v3 系列等）。

## 用量统计单位

不同类型的模型使用不同的计量单位：

| 模型类型 | 统计单位 | 说明 |
|---------|---------|------|
| 大语言模型（文本生成、深度思考、视觉理解） | Token | 按输入/输出 Token 数计费 |
| 图像生成 | 张 | 按成功生成的图像张数计费 |
| 视频生成 | 秒 | 按成功生成的视频秒数计费 |
| 语音模型 | 秒/字符/Token | 视具体模型而定 |
| 全模态模型 | Token | 文本按 Token，其他模态按对应 Token 数计费 |
| 向量模型 | Token | 按输入文本的 Token 数计费 |

> **注意**：用量数据按[业务空间](../concepts/workspace.md)维度统计，不支持按阿里云账号维度直接统计。如需查看账号级 Token 总用量，需通过费用与成本页面导出账单。

## 免费额度管理

百炼提供新用户免费额度，可在控制台的免费额度页面查看各模型的使用情况。关键配置项：

- **免费额度用完即停**：开启后，免费额度用尽时服务自动停止（返回 403 错误），避免产生额外费用。仅在账户内仍有未消耗的免费额度时可开启。

## 监控指标

模型监控提供四类核心指标，详细说明参见 [模型监控](../../raw/model-user-guide/model-monitoring/model-telemetry.md)：

- **安全**：内容安全错误次数等
- **成本**：平均单次请求调用量等
- **性能**：调用时长、首 Token 延时、RPM、TPM 等
- **错误**：失败次数、失败率、限流错误次数等

监控数据看板以卡片形式汇总模型总量、总调用次数、总失败次数、平均调用时长和平均首包时长。

## 调用日志

开通推理日志后，可查看每一次模型调用的详细记录，包括：

- 输入与输出内容（用于故障排查和内容审计）
- 本次调用的 Token 消耗
- 调用耗时

> **注意**：日志功能目前仅适用于华北2（北京）地域的部分模型。开通后从调用发生到日志记录存在分钟级延迟。

## 告警配置

模型监控支持对监控指标设置主动告警，适用于检测静默失败（如超时、Token 消耗突增）等场景：

1. **前置条件**：需先开启高级监控（在模型监控配置中开启"性能和用量指标监控"）。
2. **创建告警规则**：在模型告警页面选择要监控的模型和监控模板。
3. **通知方式**：支持短信、邮件、电话、钉钉群机器人、企业微信机器人及 Webhook。
4. **告警等级**：分为普通（INFO，仅邮件）、警告（WARNING，短信+邮件）、错误（ERROR，短信+邮件）和紧急（CRITICAL，电话+短信+邮件）。

> **注意**：告警功能目前仅适用于新加坡和华北2（北京）地域。

## 接入外部监控系统

高级监控的数据存储在私有 Prometheus 实例中，支持标准 Prometheus HTTP API，可接入 Grafana 或自建应用进行可视化分析。

主要监控指标包括：

| 指标名称 | 描述 |
|---------|------|
| `model_call_count` | 模型调用次数总和 |
| `model_call_duration` | 模型调用时长均值 |
| `model_first_token_duration` | 模型首包时长均值 |
| `model_generation_duration_per_token` | 非首包时长均值 |
| `model_usage` | 模型用量总和 |

支持通过 `workspace_id`、`model`、`apikey_id`、`status_code`、`error_code` 等标签进行过滤查询。

## 生产环境建议

- **控制输出长度**：合理设置 `max_tokens` 参数和思考长度限制，控制单次生成成本。
- **按任务选模型**：简单任务优先选择轻量级模型，避免过度使用高价模型。
- **监控与告警**：配置用量告警，及时发现异常消耗。参考 [模型用量](../../raw/model-user-guide/model-monitoring/model-usage-statistics.md) 中的应用建议。
- **优化 Prompt**：简洁清晰的 Prompt 可减少不必要的输入 Token 消耗。
- **使用批量推理**：对于非实时、大批量处理任务，批量推理通常更具成本优势。
