# 标准推理与高速推理对比

百炼平台为开发者提供了两种主要的模型推理接入方式：**标准推理（按量付费）** 和 **高速推理（TPM 预留）**。标准推理基于公共资源池，按实际 Token 用量计费，适合流量波动大或处于业务探索期的场景；高速推理通过预留专属 TPM 容量，为业务提供刚性吞吐保障，适合流量可预估且对稳定性要求高的生产环境。本文从多个维度对比两种方案，帮助开发者做出合理的技术选型。

## 核心维度对比

| 对比维度 | 标准推理（按量付费） | 高速推理（TPM 预留） |
|---------|-------------------|-------------------|
| **计费方式** | 按实际消耗的 Token 数量后付费 | 按预留的 kTPM 容量预付费（按天计费） |
| **容量保障** | 无专属保障，共享公共资源池 | 专属 TPM 容量，不与其他用户共享 |
| **限流风险** | 高峰期可能受公共池限流影响 | 预留容量内不受限流，超额部分自动降级为按量计费 |
| **模型调用方式** | 使用标准模型 ID（如 `qwen3.7-max`） | 使用系统分配的专属模型 code |
| **代码改动** | 无需改动 | 需将 `model` 参数替换为专属模型 code |
| **支持模型范围** | 覆盖文本、视觉、音频、视频、图片、3D 等全部模态 | 目前主要支持文本生成类模型（Qwen、DeepSeek、GLM、Kimi 等） |
| **API 兼容性** | [OpenAI 兼容接口](../concepts/openai-compatible.md) / DashScope API | 与标准推理接口完全兼容，仅 model 参数不同 |
| **超额处理** | 受公共池限流（可能返回 429 错误） | 超额请求自动降级为按量计费，服务不中断 |
| **扩缩容** | 无需操作，按需使用 | 支持在线扩缩容，变配期间服务不中断 |
| **缓存折扣** | 无 | 支持（如 Qwen 系列命中缓存按 20% 折算，DeepSeek-v4-pro 按 8% 折算） |
| **部署区域** | 默认区域 | 支持华北2（北京）、新加坡等多区域部署 |

## 费用模型对比

| 费用维度 | 标准推理 | 高速推理（TPM 预留） |
|---------|---------|-------------------|
| **付费方式** | 后付费 / 资源包预付费 | 预付费（按 kTPM 按天） |
| **费用可预测性** | 取决于实际用量，波动较大 | 固定费用，可精确预算 |
| **费用优化手段** | 资源包 / 节省计划 | 容量计算器估算 + 合理预留 |
| **退订机制** | 资源包不退 | 支持退订，按已使用时长 1.5 倍扣费后退款 |
| **典型价格（qwen3.7-plus，华北2）** | 按 Token 阶梯定价 | 输入 ¥2.02/kTPM/天，输出 ¥8.06/kTPM/天 |

## 运维与管理对比

| 运维维度 | 标准推理 | 高速推理（TPM 预留） |
|---------|---------|-------------------|
| **开通流程** | 开通百炼服务即可使用 | 需在控制台创建 TPM 预留并完成支付 |
| **监控能力** | 基础用量统计 | TPM 用量趋势图 + 超额降级统计 |
| **到期处理** | 无到期概念 | 到期后 2 小时内可续费；2~14 小时停服可续费；14 小时后实例删除 |
| **续费方式** | 不适用 | 手动续费 / 自动续费（到期前一天 08:00 自动扣款） |

## 适用场景建议

### 推荐使用标准推理的场景

- **业务探索期**：产品原型验证、POC 阶段，流量不确定，按量付费更灵活。
- **流量波动大**：如活动运营类业务，流量存在明显波峰波谷，预留容量利用率低。
- **多模态应用**：需要调用图片生成、视频生成、语音合成等非文本模型，当前 TPM 预留尚未覆盖这些模态。
- **成本敏感的低频场景**：日均调用量较少，按量计费的总费用低于预留最低门槛。

### 推荐使用高速推理（TPM 预留）的场景

- **生产环境核心链路**：如在线客服、智能助手等不能接受限流导致超时的业务。
- **流量可预估**：日均 / 峰值 TPM 稳定，可通过容量计算器精确预留，利用率高。
- **SLA 要求高**：对响应延迟和吞吐稳定性有明确要求的 ToB / ToC 服务。
- **预算可控**：预付费模式便于按月/按季度做成本规划。

### 混合使用策略

对于大多数中大型业务，推荐**混合使用**两种方案：将核心链路的基线流量放入 TPM 预留保障吞吐，突发流量和非核心调用走标准按量计费。TPM 预留的超额自动降级机制天然支持这种混合架构，无需额外代码适配。

## 技术选型决策参考

选择推理方案时，建议按以下优先级评估：

1. **是否需要非文本模态**：如需图片/视频/语音等模型，当前只能使用标准推理。
2. **是否能接受限流**：生产核心链路不能接受限流，应选 TPM 预留。
3. **流量是否可预估**：可预估且稳定，TPM 预留性价比更高；不可预估，按量付费更安全。
4. **预算模式偏好**：需要固定预算选预付费（TPM 预留），需要弹性选后付费（标准推理）。
