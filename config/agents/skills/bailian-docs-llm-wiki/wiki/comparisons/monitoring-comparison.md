# 应用监控与模型监控对比

百炼平台提供了两类互补的监控能力：**应用监控**（应用观测）面向百炼应用的端到端调用链路追踪与性能分析；**模型监控**面向底层模型的调用量、Token 消耗与告警。两者覆盖范围、数据来源、计费方式和适用场景均不同，开发者需要根据排障与成本治理目标选择合适的监控手段，或组合使用。

本页对比两类监控的关键维度，帮助开发者在技术选型时快速判断应使用哪一套（或两套）能力。

## 关键维度对比

| 维度 | 应用监控（应用观测） | 模型监控 |
|------|----------------------|----------|
| 监控对象 | 百炼应用（智能体应用、工作流应用、高代码应用）的完整调用链路 | 模型列表中各模型及自定义模型的调用与用量 |
| 关注层级 | 应用层（Prompt → 检索 → 模型 → 插件等 Span） | 模型层（单次模型调用的性能与消耗） |
| 核心数据来源 | 可观测链路 OpenTelemetry 服务 | 模型用量统计 + 私有 Prometheus（高级监控） |
| 数据延迟 | 分钟级 | 用量统计约 1 小时；日志分钟级延迟 |
| 关键指标 | 延时、Token 用量、调用次数、失败率、平均首 Token 耗时、平均调用时长 | Token 延时、调用时长、RPM/TPM、失败率、内容安全错误、平均单次请求调用量 |
| 链路追踪 | 支持，提供 Root Span / All Span / Model Span 三种筛选模式 | 不提供应用级链路追踪，仅记录单次模型调用 |
| 调用日志 | Trace 列表可导出 JSONL/EXCEL；支持按 Request ID/Trace ID/Span ID 搜索 | 推理日志记录输入输出内容、Token 消耗、耗时 |
| 告警能力 | 暂未提供主动告警，仅控制台查看 | 支持主动告警，可配置短信/邮件/电话/钉钉/企业微信/Webhook，分级 INFO/WARNING/ERROR/CRITICAL |
| 数据导出 | 支持 JSONL、EXCEL 导出 | 支持接入 Grafana 或自建系统（Prometheus HTTP API） |
| 外部系统集成 | 仅控制台操作，暂无 API 接口 | 高级监控数据存储于私有 Prometheus，提供标准 HTTP API |
| 计费方式 | 应用观测功能本身免费，但需支付 OpenTelemetry 服务的存储费用 | 监控功能本身不额外收费；用量按模型类型计量（Token/张/秒/字符） |
| 数据标注与评测 | 支持对 Span 标注标签，可将线上数据导入评测集（最多 50 个字段） | 不支持直接标注与导入评测集 |
| 支持应用/模型范围 | 智能体应用、工作流应用、高代码应用（Assistant API 创建的智能体暂不支持） | 用量统计支持所有模型；普通监控支持所有模型；高级监控/告警/日志有地域限制 |
| 地域限制 | 依赖 OpenTelemetry 服务可用性 | 高级监控支持北京、新加坡、弗吉尼亚；告警支持北京、新加坡；日志仅北京、新加坡部分模型 |
| 典型场景 | 排查应用链路中哪个环节延时高、Token 消耗异常、检索效果差 | 监控模型调用量趋势、设置成本告警、接入外部 Grafana 做可视化 |

## 适用场景建议

### 优先使用应用监控（应用观测）的场景

- 需要追踪**智能体应用或工作流应用**的完整处理流程，定位是检索、重写、模型调用还是插件环节导致延时或失败。
- 需要查看**具体某次请求**的 Prompt 内容与输出、Token 用量，并通过 Request ID/Trace ID/Span ID 精确检索。
- 需要将线上调用数据**导入评测集**做后续评测，或对 Span 做自定义标签管理。
- 应用包含向量生成、向量检索、大模型调用等多环节，需要端到端的可观测性。

### 优先使用模型监控的场景

- 关注**模型层面**的调用量、Token 消耗趋势与成本治理，需要按[业务空间](../concepts/workspace.md)维度统计各模型用量。
- 需要**主动告警**能力，例如静默失败、Token 消耗突增、失败率升高时及时通知（短信/电话/钉钉等）。
- 需要将监控数据接入**外部 Grafana 或自建系统**做长期可视化与跨系统分析（通过 Prometheus HTTP API）。
- 需要查看**推理日志**进行内容审计或故障排查（限北京、新加坡部分模型）。
- 需要管理**免费额度**，开启"免费额度用完即停"避免额外费用。

### 组合使用建议

对于生产环境的百炼应用，建议同时启用两类监控：

- 用**应用监控**做应用链路级排障，快速定位问题环节。
- 用**模型监控**做模型用量治理与主动告警，控制成本并及时发现异常。

两者数据互补：应用监控的 Model Span 可与模型监控的调用记录关联，实现从应用入口到模型底层的全链路可观测。注意应用监控暂无 API 接口，自动化场景需依赖模型监控的 Prometheus API。
