# model compression

模型压缩是百炼平台提供的量化能力，可将全精度微调模型转换为低精度版本，从而降低推理时的显存占用、缩减部署规格和成本，同时提升推理吞吐。压缩产出的模型可直接用于模型部署。详细介绍参见[模型压缩](../../raw/model-user-guide/model-compression/model-compression-introduction.md)。

## 适用场景

当你通过模型调优产出了全精度自定义微调模型，但部署成本较高时，可通过模型压缩将其量化到更小的部署单元规格。例如，qwen3.5-flash-2026-02-23 的微调模型压缩前需要 MU1 * 2（108 元/小时），压缩后仅需 MU8 * 1（47 元/小时）。

## 支持压缩的模型

当前支持以下基础模型对应的自定义微调模型：

| 模型系列 | 模型名称 | 压缩前部署 | 压缩后部署 |
| --- | --- | --- | --- |
| 千问 | qwen3.5-flash-2026-02-23 | MU1 * 2（108 元/小时） | MU8 * 1（47 元/小时） |

> 具体支持的模型以百炼控制台「模型压缩」页面为准。

## 前提条件

- 当前工作空间内至少有一个支持压缩的自定义微调模型（通过模型调优产出）。
- 已量化的模型不支持二次压缩。

## 操作流程

1. 登录百炼控制台，进入 **模型 > 模型训练 > 模型压缩** 页面。
2. 点击 **+ 创建压缩任务**，填写以下配置：
   - **任务名称**（必填）和**任务描述**（可选）。
   - **选择源模型**：从当前工作空间内符合条件的自定义微调模型中选择。
   - **量化产出模型名后缀**：用于拼接产出模型 ID，仅支持小写字母和数字，长度不超过 8 位。
   - **量化模板**：模板名称中包含目标部署规格（如 MU5、MU8），选择模板即决定压缩后可部署的规格。
   - **校准数据**（部分模板需要）：用于统计模型权重和激活值的数值分布，以确定量化参数。需从已发布数据集中选择，OSS 挂载的数据集暂不支持。
3. 点击 **开始压缩**，等待任务完成。
4. 任务成功后，在任务列表中点击压缩后的模型名称，跳转到 **我的模型** 页面进行部署。

关于校准数据的详细要求，参见[模型压缩](../../raw/model-user-guide/model-compression/model-compression-introduction.md)中的校准数据说明。

## 任务状态说明

压缩任务包含以下状态：待开始、排队中、运行中、停止中、压缩成功、压缩失败、已取消。

- 仅「排队中」或「运行中」状态的任务支持取消。
- 终态任务（压缩成功 / 压缩失败 / 已取消）可删除。
- 删除任务记录仅删除任务本身，已生成的产出模型保留在 **我的模型** 中。

## 限制与注意事项

- **不可逆操作**：压缩产出的模型不支持继续训练，也不支持二次压缩。如需迭代，请回到上游全精度模型重新训练后再压缩。
- **部署规格**：压缩后模型的部署单元规格由所选量化模板决定，部署数量在 **模型部署** 中配置。
- **校准数据限制**：控制台 UI 仅支持「内部上传」类型且「已发布」状态的数据集；OpenAPI 调用可通过 `custom_calibration_file_ids` 字段传入数据集组 ID。

更多常见问题和操作细节，参见[模型压缩原始文档](../../raw/model-user-guide/model-compression/model-compression-introduction.md)。

## 计费

模型压缩功能当前限时免费，具体截止时间以控制台公告为准。压缩后模型的部署费用按所选部署单元的标准规格计费。
