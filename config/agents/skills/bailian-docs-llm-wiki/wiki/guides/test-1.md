# test 1

阿里云百炼的成本管理由免费额度、模型调用计费、模型训练与部署计费、账单查询、节省计划与资源包五个环节构成。本文汇总这些环节的关键规则与操作路径，帮助开发者在调用模型前预估费用、调用后核对账单、并选择合适的优惠方案以降低成本。

## 免费额度

首次开通阿里云百炼后，平台会自动为各模型发放新人专属免费额度，无需手动领取。免费额度仅抵扣模型**实时推理**费用，不抵扣 Batch 调用、模型调优、模型部署及自定义模型的费用。详见 [新人免费额度](../../raw/model-user-guide/test-1/new-free-quota.md)。

**有效期**：自开通或模型申请通过之日起 30~90 天。自 2025 年 9 月 8 日 11 点起，首次开通的用户获赠的免费额度有效期统一调整为 90 天；此前已开通的用户不受影响。额度到期或耗尽后，继续调用将按模型调用价格计费。

**适用范围**：仅华北 2（北京）地域且服务部署范围为「中国内地」的模型享有免费额度，其他地域和部署范围无免费额度。

**共享与查询**：阿里云主账号与其 RAM 子账号共享免费额度。可在控制台的「模型用量 > 免费额度」页签，或「模型广场」模型详情页的「免费额度」区域查看余量。控制台显示为分钟级更新（需手动刷新）。

**免费额度用完即停**：为防止意外扣费，可开启「免费额度用完即停」开关。开启后额度耗尽时调用返回错误码 `AllocationQuota.FreeTierOnly`，不再扣费。全新未认证用户额度耗尽后默认无法继续调用，需完成实名认证并充值后方可按量付费。

> **注意**：账户欠费时，即使其他模型仍有免费额度也无法调用，需先结清欠费。

## 模型调用计费（按量付费）

模型调用默认按量计费，单价以控制台「模型广场」展示为准，完整规则见 [模型调用价格](../../raw/model-user-guide/test-1/model-pricing.md)。

**阶梯计费**：部分模型按单次请求的输入 Token 总量分阶梯计价，该请求的所有 Token 均按对应阶梯单价结算（不是分段累计）。例如某模型设 0<Token≤32K 与 32K<Token≤128K 两档，输入 100K Token 时全部 Token 按 32K<Token≤128K 档单价结算。

**计费维度**：按输入 Token 和输出 Token 分别计费。华北 2（北京）价格最低，新加坡、德国（法兰克福）、日本（东京）等海外地域价格更高（服务部署范围为「国际」或「欧盟」）。

**影响价格的因素**：

- **Batch 调用**：若模型支持，其输入和输出 Token 单价均按实时推理价格的 50% 计费。
- **上下文缓存**：若模型支持，仅输入 Token 享有折扣。
- 两者不能同时生效。

**示例**（华北 2，中国内地，每百万 Token）：

| 模型 | 输入区间 | 输入单价 | 输出单价 |
| --- | --- | --- | --- |
| qwen3.7-max（思考+回答） | 0<Token≤1M | 12 元 | 36 元 |
| qwen3-max（非思考+思考） | 0<Token≤32K / 32K<Token≤128K / 128K<Token≤256K | 2.5 / 4 / 7 元 | 10 / 16 / 28 元 |
| qwen3-max-2025-09-23（仅非思考） | 0<Token≤32K / 32K<Token≤128K / 128K<Token≤256K | 6 / 10 / 15 元 | 24 / 40 / 60 元 |
| qwen3.7-plus（思考+回答） | 0<Token≤256K / 256K<Token≤1M | 2 / 6 元 | 8 / 24 元 |
| qwen3.6-plus（思考+回答） | 0<Token≤256K / 256K<Token≤1M | 2 / 8 元 | 12 / 48 元 |
| qwen-max（仅非思考，无阶梯） | — | 2.4 元 | 9.6 元 |

各模型在华北 2（北京）享有 100 万 Token 免费额度（有效期 90 天）；海外地域无免费额度。

## 模型训练与部署计费

模型训练和部署属于独立计费项，不支持用免费额度抵扣。详见 [模型训练与部署计费](../../raw/model-user-guide/test-1/model-training-and-deployment-billing.md)。

### 模型训练

计费方式为按训练 Token 计费。

- **文本生成模型（千问）**：`费用 = (训练数据 Token 总数 + 混合训练数据 Token 总数) × 循环次数 × 训练单价`，最小计费单位 1 Token。例如 Qwen3-8B 训练单价 ¥0.006/千 Token，Qwen3-32B ¥0.04/千 Token，Qwen2.5-72B-Instruct ¥0.15/千 Token。
- **图像生成模型（万相）**：`费用 = 训练 Token 总量 × 训练单价`，训练 Token 总量 ≈ `max_steps × Lstep`（Lstep 近似等于 Lmax，Lmax 由 generation_type 与 max_token_length 决定）。例如 wan2.7-image-pro 单价 0.08 元/千 Token。
- **视频生成模型（万相）**：训练 Token 总量 = `(∑ 视频计费时长) × (max_pixels/1024) × n_epochs`。wan2.5 单条视频计费时长最多按 10 秒，wan2.2 最多按 5 秒计算。

训练完成后的新模型须先完成**模型部署**，才能评测和调用。

### 模型部署

部署计费有两种模式：

**1. 按使用时长计费（预置吞吐）**：`费用 = 使用时长 × (输入 TPM 单价 × 输入 TPM + 输出 TPM 单价 × 输出 TPM)`。后付费按小时计费、预付费按天计费。当输入超过最长输入 Token 或超出购买的 TPM 量时，调用自动切换为按量付费模式，响应 Header 会包含 `x-dashscope-ptu-overflow:true`。预付费订单到期后延后 2 小时停止服务，停止后资源保留 14 小时后释放；后付费账户欠费时资源保留并继续计费 24 小时后自动释放。

**2. 按使用时长计费（模型单元）**：`费用 = 使用时长(小时) × 模型单元数量 × 模型单元单价`，最小计费单位为分钟。预付费按月计费时公式为 `包月数 × 模型单元数量 × 月单价`。预付费首月提前退订，日单价按 1.2 倍计费。模型单元分为 Instruct（非思考模式）、Thinking（思考模式）两种类型；部署类型支持 PD 分离模式，可降低首 Token 延迟、提高吞吐。

## 账单查询与成本管理

详见 [账单查询与成本管理](../../raw/model-user-guide/test-1/bill-query-and-cost-management.md)。

**出账时延**：大模型推理分钟级出账（通常 2~10 分钟），批量推理、模型训练、知识库等小时级出账。公测、邀测模型不产生账单记录。

**查询入口**：

- **费用概览**：百炼控制台「模型 > 用量 & 费用 > 费用概览」，展示当月总消费（仅含大模型推理相关费用），拆分为订阅购买费用与账单费用。可按模型或 API Key 筛选，对比支出趋势，设置月度费用告警。
- **账单详情**：阿里云用户中心「账单详情」页面，产品名称选择「大模型服务平台百炼」后下载账单。

**实例 ID 字段解读**：账单「实例 ID（出账粒度）」以英文分号分隔，格式为 `ApiKeyID;业务空间ID;模型名称;输入/输出类型;调用渠道;免费额度用完即停标识`。若不含 ApiKeyID，通常表示该费用来自控制台调用。账单「计费项」统一显示为「大模型文本消耗量」，区分具体模型需看实例 ID 中紧跟[业务空间](../concepts/workspace.md) ID 的字段。调用渠道取值：`app`（代码调用）、`bmp`（控制台模型体验）、`assistant-api`（Assistant API）。

**分账管理**：给[业务空间](../concepts/workspace.md)绑定标签后可按部门或项目归集费用。在「标签管理」页面将标签绑定到[业务空间](../concepts/workspace.md)的 Workspace ID，然后在「费用标签」页面启用该标签。配置完成后分账账单 T+1 天生效。

**欠费判定**：账户可用额度 < 0 视为欠费，公式为 `可用额度 = (现金余额 + 信控额度) - (当月未结清 + 历史未结清)`。欠费按账单「商品名称」维度判定影响：仍有免费额度/节省计划/资源包额度时可继续使用，已购 Coding Plan 或 Token Plan 套餐额度独立于账户余额欠费期间可继续使用但会导致自动续费失败；以上额度均无则该商品下服务暂停。

**按量付费扣款模式**：采用「预占 + 月结」——系统先冻结部分额度，月账期结束后（次月初）生成最终账单并实际扣款，并非实时扣款。

> **注意**：百炼的联网搜索等附加功能按调用次数单独计费（后付费），与模型推理费用分开出账。即使近期未操作控制台，历史创建的应用或代码中若开启了 `enable_search`，每次被调用仍会产生联网搜索费用，可能导致「没怎么用却欠费」。

## 节省计划与资源包

为降低按量付费成本，百炼提供节省计划与资源包两类优惠方案。详见 [节省计划与资源包](../../raw/model-user-guide/test-1/savings-plan-and-resource-package.md)。

### AI 通用型节省计划（推荐）

通过承诺每月消费金额换取阶梯式折扣，最高 5.3 折。可抵扣「阿里直供」的全部模型，灵活性最高，是绝大多数场景下的首选。

- **承诺周期**：3 / 6 / 12 / 24 个月，以月为单位。月承诺周期结束时剩余额度自动过期，不可累积到下一周期（3 个月承诺 ≠ 3 个月总额度，而是每月独立获得额度）。
- **月承诺金额**：1000 元起，以 10 元为单位调整，不设上限。
- **付费方式**：全预付（一次性支付整个承诺周期金额，折扣最大）或零预付（按月支付，需联系商务经理开通白名单）。
- **抵扣范围**：支持模型调用（输入和输出 Tokens）、模型原生工具调用（Function Call、网页抓取等）、上下文缓存、批量推理；不支持抵扣模型调优、模型部署费用。
- **抵扣顺序**：免费额度 > 资源包 > 其他模型节省计划 > AI 通用型节省计划 > 按量付费。
- **适用地域**：华北 2（北京）、美国（弗吉尼亚）、新加坡、德国（法兰克福）。
- **模型分档**：A 类（千问、开源、向量、排序、行业模型、原生工具调用）、B 类（图像生成、语音、视频）、C 类（qwen3.6-max-preview、DeepSeek、Kimi、GLM、MiniMax、HappyHorse）。三方直供模型不支持抵扣。承诺金额越高、周期越长折扣越大，例如全预付、月承诺 100 万+、24 个月、A 类模型可享 5.3 折。

### 其他模型节省计划

适合用量较小或集中于特定模型的场景。购买后立即生效，有效期根据套餐而定，到期后剩余金额作废不支持退款。仅支持抵扣模型调用费用（输入和输出 Tokens），不支持抵扣工具调用、上下文缓存、批量推理、模型调优、模型部署。包括：

- **大语言模型推理节省计划**：11 档（20~500,000 元），适用华北 2（北京），覆盖以 Token 计价的文本生成与多模态模型。各档位均无折扣，按模型调用价格扣费。不支持向量模型和排序模型。
- **千问语音模型节省计划**：5 档（20~5,000 元），最高 8 折。
- **向量及排序模型服务节省计划**：5 档（100~10,000 元），最高 7 折。

### 资源包

预先购买具体 Token 数量，用于抵扣特定模型超出免费额度后的实时推理用量。购买后立即生效，有效期根据套餐而定，到期剩余 Token 作废。抵扣顺序与节省计划一致（免费额度 > 资源包 > …）。已使用部分无法退款，未使用部分可按未使用额度费用申请退款。例如大语言模型推理资源包（qwen-plus / qwen-max / qwen-turbo），仅支持抵扣单次请求输入在 0<Token≤128K 阶梯的实时推理费用（非思考模式），不支持抵扣 Token>128K 阶段的费用。

## 选型建议

1. **优先评估 AI 通用型节省计划**：覆盖模型最广、折扣力度最大，承诺金额越高、周期越长折扣越大。
2. **调用集中且稳定的特定模型**：可补充对应的资源包或其他模型节省计划，按抵扣顺序叠加使用。
3. **海外地域调用**：免费额度不适用，价格更高，需独立评估节省计划在海外地域的折扣档位（选择对应的全局计费项查询）。
4. **设置预警**：在「高额消费预警」页面设置日账单阈值，在「费用概览」设置月度费用告警，在「我的试用」配置免费额度余量预警（默认余量降至 20% 或耗尽时通知），避免欠费停服。
