# model high speed inference

TPM 预留是百炼平台提供的专属推理容量保障方案，通过为指定模型锁定独占的 TPM（Tokens Per Minute）吞吐量，确保业务高峰期不受公共资源限流影响。超出预留容量的请求会自动降级为按量计费处理，服务不中断。

## 功能特性

- **容量保障**：预留的 TPM 容量为业务专属，不与其他用户共享。
- **专属模型 code**：创建 TPM 预留后系统自动生成专属模型 code，需将 API 请求中的 `model` 参数替换为该 code。
- **超额不中断**：超出预留容量的请求自动降级为按量计费，无需修改代码。

详细功能说明参见 [TPM 预留](../../raw/model-user-guide/model-high-speed-inference/tpm-reservation.md)。

## 方案对比与选型

百炼提供多种推理容量与计费方案，开发者应根据业务特征选择合适方案：

| 方案 | 计费单位 | 容量保障 | 适用场景 | 超额处理 | 代码改动 |
|------|---------|---------|---------|---------|---------|
| 按量付费 | 按 token | 无（共享公共池） | 流量波动大/短期 | 受公共限流 | 无需改动 |
| 资源包/节省计划 | 预付费额度 | 承诺用量折扣（非专属） | 费用优化 | 超出转按量 | 无需改动 |
| TPM 预留 | 按 kTPM 预付费 | 专属容量刚性兑付 | 流量可预估、不能接受限流 | 超出降级公共池按量 | 替换 model 参数 |
| PTU 模型部署 | 按 kTPM 预付费 | 专属部署实例 | 高吞吐高性能 | 超出转按量 | 替换 model 参数 |

如果业务流量可预估且对限流敏感，推荐使用 TPM 预留；若需要更高性能保障，可考虑 PTU 专属部署。

## 容量换算参数

部分模型支持长输入阶梯系数和缓存折扣，影响实际容量消耗：

| 模型 | 输入长度上限 | 缓存折扣 | 长输入阶梯系数 |
|------|------------|---------|--------------|
| Qwen 系列 | 256K | 0.2（命中部分按 20% 折算） | 无阶梯（1.0） |
| deepseek-v4-pro | 256K | 0.08（命中部分按 8% 折算） | 无阶梯（1.0） |

创建页面右侧的**容量计算器**可根据每分钟请求数（RPM）、平均输入/输出长度、预估缓存命中率自动估算所需 TPM 额度。

## 创建与接入

1. 登录百炼控制台 TPM 预留页面，单击**创建 TPM 预留**。
2. 填写预留名称、选择模型、设定输入/输出 TPM（单位 kTPM）、购买时长等参数。
3. 确认费用后单击**立即购买**完成支付。
4. 在详情页**概览** Tab 复制**专属模型 code**。
5. 将 API 请求中的 `model` 参数替换为该 code。

Python 示例：

```python
import dashscope

response = dashscope.Generation.call(
    api_key="your-api-key",
    model="your-dedicated-model-code",   # 替换为专属模型 code
    messages=[{"role": "user", "content": "你好"}],
)
print(response.output.text)
```

> **注意**：短时间内请求量快速拉升时，系统需要短暂预热以匹配算力，预热期间部分请求可能出现延迟波动。建议做好请求排队或重试机制。

## 管理操作

- **扩缩容**：在详情页调整输入/输出 TPM，变配期间服务不中断。
- **续费**：手动续订或开启到期自动续费（到期前一天 08:00 自动扣款）。
- **退订**：通过费用中心完成退订，退订后专属模型 code 失效，不可恢复。

缩容与退订退费公式：`退款 = 降量部分预付费 - (降量部分预付费 × 已用时长/购买时长 × 1.5)`

## 计费与生命周期

- 部署成功即开始计费，预留容量内的调用不额外收费。
- 预付费一次性支付，从购买成功起连续生效，详细费用以百炼控制台为准。
- 超出保障额度后自动降级为标准按量计费，可在详情页**超额降级统计**中查看降级情况。

实例生命周期如 [TPM 预留](../../raw/model-user-guide/model-high-speed-inference/tpm-reservation.md) 所述：到期后 2 小时内仍可调用和续费；2~14 小时后停止服务但可续费；14 小时后实例删除不可恢复。

## 常见问题

**超出预留容量时会怎样？** 超额请求自动降级为按量计费，服务不中断。频繁降级时建议扩容。

**如何判断是否需要扩容？** 在详情页查看 TPM 用量趋势图和超额降级统计，利用率持续接近 100% 或频繁降级时应扩容。

**预留到期后会怎样？** 专属模型 code 失效，后续请求回退到公共资源按量计费。建议提前开启自动续费。
