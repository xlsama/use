# 业务空间

业务空间（Workspace）是阿里云百炼平台进行资源组织、权限管控和配额管理的最小隔离单元。所有应用组件 API 调用、模型调用、知识库、Prompt 模板等操作都必须在某个业务空间的上下文中进行。

## 核心特性

### 地域隔离

单个业务空间不能跨地域存在，每个地域（如华北2北京、新加坡、弗吉尼亚）下的默认业务空间也是相互独立的。跨地域业务需分别在不同地域的业务空间中操作。

### 管理角色

百炼的权限体系基于三种角色，围绕业务空间展开：

| 角色 | 权限范围 |
|------|---------|
| 超级管理员 | 跨空间统一管理用户权限、模型授权与限流、API Key |
| 业务空间管理员 | 仅管理当前空间内的用户权限和 API Key |
| 普通用户 | 使用被授权的空间、页面和资源 |

### 默认空间的限制

默认业务空间无法限制模型调用、调优和部署，所有模型均可使用且无法设置限流。若需要精细化的模型调用控制，需创建非默认业务空间。

## 在百炼各场景中的使用

### 应用组件 API 调用

应用组件 API（`bailian/2023-12-29`）的所有接口（数据连接、知识库、Prompt 模板、长期记忆等）均需在业务空间上下文中调用。RAM 授权粒度为操作级，RAM 代码为 `sfm`。

### 权限管理

业务空间是精细化权限管理的承载单元，可管理的权限维度包括：

1. **模型调用限制**：控制特定模型能否在该空间调用（控制台和 API），并设置 QPM 和 Token 限流。
2. **模型训练限制**：控制模型能否在该空间进行调优及调优后部署。
3. **模型部署限制**：控制模型能否在该空间直接部署。
4. **用户控制台权限**：管理 RAM 用户可使用的控制台功能页面（不影响 API Key 调用）。

### API Key 归属

- 单个 API Key 只能归属一个地域内的一个业务空间和一个用户，不可转移。
- API Key 的可调用功能和模型限流与归属业务空间的权限一致，不受用户控制台权限影响。
- 自 2026 年 3 月 25 日起，华北2（北京）地域新创建的 API Key 均归属主账号。

### 监控与用量统计

- 模型用量按业务空间维度统计各模型的调用量和 Token 消耗，数据延迟约 1 小时。
- 用量数据不支持按阿里云账号维度直接统计，如需账号级 Token 总用量需通过费用与成本页面导出账单。
- Prometheus 监控指标可通过 `workspace_id` 标签进行过滤查询。

### 应用观测

应用观测追踪的是业务空间内百炼应用的完整处理链路。若在应用列表中找不到已创建的应用，常见原因之一是应用不属于当前业务空间。

## 关键配置

### OpenAPI 接口权限

RAM 用户默认无权调用百炼应用的 Open API。需阿里云主账号在 RAM 控制台授予以下策略之一：

- `AliyunBailianDataFullAccess`：可调用应用 API 目录下的所有接口。
- `AliyunBailianDataReadOnlyAccess`：仅可调用只读类接口。

### 生产环境空间规划

推荐按环境或业务线划分业务空间，实现隔离与成本管理：

- **按环境划分**：为 dev、test、prod 创建独立空间，例如 `project-dev-workspace`、`project-prod-workspace`。
- **按业务线划分**：为不同部门创建独立空间，便于权限和成本核算。

### 限流配额分配

将主账号总配额按比例分配给各业务空间并预留缓冲。例如总配额 1000 QPM：

| 空间 | 配额 | 占比 |
|------|------|------|
| 生产空间 | 600 QPM | 60% |
| 测试空间 | 200 QPM | 20% |
| 开发空间 | 100 QPM | 10% |
| 预留缓冲 | 100 QPM | 10% |

## 注意事项

- 删除知识库前需先解除与应用的关联（仅支持控制台操作），删除知识库不会删除数据连接中已导入的文件。
- 将 RAM 账号移出业务空间后，其 API Key 会失效，重新加入后恢复；在 RAM 控制台删除账号/角色则对应 API Key 永久失效。
- 华北2（北京）地域的 API Key 支持设置 IP 访问白名单。
